## 💡 什么是配对？

### 类比：配对就像"访客登记"

```

│  │     ├─ 批准 → 发放访客证 → 允许进入                             │   │
│  │     └─ 拒绝 → 拒绝进入                                        │   │

│  │  系统生成配对码                                                   │   │

```

### 配对的两个应用场景

| 场景 | 说明 | 使用方式 |
|------|------|----------|
| **私信配对** | 控制谁可以与 AI 机器人对话 | `openclaw pairing approve` |
| **节点配对** | 控制哪些设备可以连接到网关 | `openclaw devices approve` |

## 📨 私信配对

### 工作机制

当渠道配置为 `pairing` 私信策略时，未知发送者会收到一个配对码，其消息**不会被处理**，直到你批准。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       私信配对流程                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  未知用户发送消息                                                      │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  1. 检查发送者状态                                              │   │
│  │     • 已在白名单？ → 直接处理                                    │   │
│  │     • 已在黑名单？ → 拒绝                                        │   │
│  │     • 未知用户？ → 进入配对流程                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  2. 生成配对码                                                  │   │
│  │     • 8 个字符，大写字母                                        │   │
│  │     • 无歧义字符（排除 0O1I）                                   │   │
│  │     • 1 小时后过期                                              │   │
│  │     • 每个渠道最多 3 个待处理请求                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  3. 发送配对码给用户                                            │   │
│  │     • 用户收到配对码消息                                        │   │
│  │     • 消息说明需要配对才能继续                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  4. 等待管理员审批                                              │   │
│  │     • 管理员运行 list 查看待处理请求                            │   │
│  │     • 管理员批准或拒绝                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 配对码规则

| 属性 | 值 | 说明 |
|------|-----|------|
| **长度** | 8 个字符 | 固定长度 |
| **格式** | 大写字母 | 无歧义字符 |
| **排除** | `0O1I` | 容易混淆的字符 |
| **有效期** | 1 小时 | 超时自动过期 |
| **并发限制** | 每渠道 3 个 | 额外请求被忽略 |

### 管理命令

```bash
# 查看待处理的配对请求
openclaw pairing list telegram
openclaw pairing list whatsapp
openclaw pairing list discord

# 批准配对
openclaw pairing approve telegram <CODE>

# 拒绝配对
openclaw pairing reject telegram <CODE>

# 查看已批准的用户
openclaw pairing approved

# 查看已拒绝的用户
openclaw pairing rejected
```

### 支持的渠道

| 渠道 | 支持状态 |
|------|----------|
| Telegram | ✅ |
| WhatsApp | ✅ |
| Signal | ✅ |
| iMessage | ✅ |
| Discord | ✅ |
| Slack | ✅ |

### 存储位置

配对状态存储在 `~/.openclaw/credentials/` 下：

| 文件 | 内容 |
|------|------|
| `<channel>-pairing.json` | 待处理的配对请求 |
| `<channel>-allowFrom.json` | 已批准的白名单 |

> **🔐 安全提醒**：这些文件控制对你助手的访问权限，请确保只有你本人可以访问。

## 📱 节点设备配对

### 工作机制

节点设备（iOS/Android/macOS）作为 `role: node` 连接到 Gateway 网关时，需要经过设备配对批准。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       节点配对流程                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  节点设备启动                                                          │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  1. 连接到 Gateway 网关                                        │   │
│  │     • 建立 WebSocket 连接                                       │   │
│  │     • 发送设备信息                                              │   │
│  │     • 请求配对                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  2. Gateway 创建配对请求                                       │   │
│  │     • 生成唯一的请求 ID                                         │   │
│  │     • 生成配对令牌                                              │   │
│  │     • 等待管理员批准                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  3. 管理员审批                                                  │   │
│  │     • 查看待处理设备                                            │   │
│  │     • 验证设备身份                                              │   │
│  │     • 批准或拒绝                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  4. 配对完成                                                    │   │
│  │     • 设备获得访问权限                                          │   │
│  │     • 可以调用节点功能                                          │   │
│  │     • 定期心跳保持连接                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 管理命令

```bash
# 查看待处理的设备请求
openclaw devices list

# 批准设备
openclaw devices approve <requestId>

# 拒绝设备
openclaw devices reject <requestId>

# 查看已配对设备
openclaw devices paired

# 查看设备详情
openclaw devices status <deviceId>
```

### 存储位置

节点配对状态存储在 `~/.openclaw/devices/` 下：

| 文件 | 内容 |
|------|------|
| `pending.json` | 待处理的设备请求（短期，会过期） |
| `paired.json` | 已配对设备列表 + 令牌 |

### 配对状态

| 状态 | 说明 |
|------|------|
| **pending** | 等待管理员审批 |
| **approved** | 已批准，设备已配对 |
| **rejected** | 已拒绝 |
| **expired** | 请求已过期 |

## 🔒 安全考虑

### 为什么需要配对？

| 威胁 | 没有配对 | 有配对 |
|------|----------|--------|
| **垃圾消息** | 无法阻止 | 直接拒绝 |
| **提示词注入** | 可能成功 | 需要审批 |
| **资源滥用** | 无法控制 | 可限制访问 |
| **隐私泄露** | 风险高 | 可控访问 |

### 最佳实践

│                       配对安全最佳实践                                     │
├─────────────────────────────────────────────────────────────────────────┤

│  │  • 使用 pairing 作为默认私信策略                                │   │
│  │  • 只批准你认识的用户                                          │   │
│  │  • 定期审查白名单                                              │   │

### 文件权限检查

```bash
# 检查配对文件权限
ls -la ~/.openclaw/credentials/*-pairing.json
ls -la ~/.openclaw/credentials/*-allowFrom.json

# 检查节点配对文件权限
ls -la ~/.openclaw/devices/

# 设置安全权限（如果需要）
chmod 600 ~/.openclaw/credentials/*-pairing.json
chmod 600 ~/.openclaw/credentials/*-allowFrom.json
chmod 600 ~/.openclaw/devices/*.json
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 配对码无效 | 已过期或错误 | 等待用户重新请求 |
| 无法批准 | 命令格式错误 | 检查渠道名称和配对码 |
| 设备无法配对 | 网关未运行 | 检查网关状态 |
| 配对后仍无响应 | 需要重启渠道 | 重启对应渠道 |

### 调试命令

```bash
# 查看配对状态
openclaw pairing list --verbose

# 查看设备状态
openclaw devices list --verbose

# 检查配对日志
openclaw logs --lines 100 | grep "pairing"
```
