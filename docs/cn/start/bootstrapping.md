## 💡 什么是引导？

### 类比：引导就像"入职登记"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    入职登记类比                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：新员工入职                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第一步：发放工牌和手册                                          │   │
│  │  第二步：填写个人信息表                                         │   │
│  │  第三步：了解公司规章制度                                       │   │
│  │  第四步：完成入职，开始工作                                     │   │
│  │                                                                 │   │
│  │  特点：                                                          │   │
│  │  • 只执行一次                                                   │   │
│  │  • 建立员工档案                                                 │   │
│  │  • 后续基于档案工作                                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 场景：智能体首次启动                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第一步：创建工作区目录                                         │   │
│  │  第二步：写入种子文件（模板）                                   │   │
│  │  第三步：问答式收集身份信息                                     │   │
│  │  第四步：生成身份文件，完成引导                                 │   │
│  │                                                                 │   │
│  │  特点：                                                          │   │
│  │  • 首次运行自动触发                                             │   │
│  │  • 建立智能体"人格"                                             │   │
│  │  • 后续基于配置运行                                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **引导** | 首次运行时的初始化仪式 |
| **工作区** | 存放智能体配置的目录（默认 `~/.openclaw/workspace`） |
| **种子文件** | 初始化时创建的模板文件 |
| **身份文件** | 描述智能体"人格"的配置 |

### 触发时机

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       引导触发时机                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 完成向导配置（onboarding）                                          │
│         │                                                               │
│         ▼                                                               │
│  2. 首次启动智能体                                                      │
│         │                                                               │
│         ▼                                                               │
│  3. 检测工作区是否存在引导文件                                          │
│         │                                                               │
│         ▼                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  存在 BOOTSTRAP.md？                                             │   │
│  │                                                                 │   │
│  │  YES → 执行引导流程                                              │   │
│  │  NO  → 跳过，正常启动                                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

```

│                       引导执行步骤                                         │

│  │  • USER.md - 用户描述                                           │   │
│  │  • SOUL.md - 行为特征                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  步骤 4：清理标记                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  删除 BOOTSTRAP.md                                                │   │
│  │  确保引导只执行一次                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📁 种子文件说明

### 文件结构

```
~/.openclaw/workspace/
├── skills/                    # 自定义技能目录
├── AGENTS.md                  # 系统提示词模板
├── BOOTSTRAP.md               # 引导触发器（执行后删除）
├── IDENTITY.md                # 身份描述模板
├── USER.md                    # 用户信息模板
├── SOUL.md                    # 行为特征（生成）
├── TOOLS.md                   # 工具配置
└── [自定义文件]               # 用户添加的内容
```

### 各文件作用

| 文件 | 来源 | 作用 |
|------|------|------|
| `BOOTSTRAP.md` | 自动生成 | 触发引导流程，完成后删除 |
| `AGENTS.md` | 模板种子 | 定义智能体的系统行为 |
| `IDENTITY.md` | 问答生成 | 描述智能体的身份和角色 |
| `USER.md` | 问答生成 | 描述用户的特点和偏好 |
| `SOUL.md` | 问答生成 | 定义智能体的性格特征 |

### BOOTSTRAP.md 内容

```markdown
# 🎯 智能体引导

欢迎使用 OpenClaw！让我们来设置你的智能体身份。

## 问答仪式

请回答以下问题（一次一个）：

1. 你希望智能体叫什么名字？
2. 智能体的主要角色是什么？
3. 智能体应该有什么样的个性？
4. 有什么特殊偏好需要配置吗？

---

> 回答完成后，这些信息将用于创建你的智能体配置文件。
```

## 🌐 远程网关场景

### 运行位置

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       远程网关引导场景                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  本地 Mac 应用                    远程服务器                              │
│  ┌─────────────────┐         ┌─────────────────────────────────┐       │
│  │ macOS App       │         │ Gateway 服务                     │       │
│  │                 │         │                                 │       │
│  │ 连接到远程网关   │────────►│ 工作区在这里创建：               │       │
│  │                 │         │ ~/.openclaw/workspace/           │       │
│  │                 │         │                                 │       │
│  │                 │         │ 引导在这里执行：                 │       │
│  │                 │         │ • 问答交互                       │       │
│  │                 │         │ • 文件写入                       │       │
│  │                 │         │ • 配置生成                       │       │
│  └─────────────────┘         └─────────────────────────────────┘       │
│                                                                         │
│  重要提示：                                                              │
│  • 工作区文件在远程服务器上                                             │
│  • 需要在服务器上编辑配置文件                                            │
│  • 例如：user@gateway-host:~/.openclaw/workspace/                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 远程场景注意事项

| 场景 | 工作区位置 | 编辑方式 |
|------|-----------|----------|
| **本地网关** | `~/.openclaw/workspace` | 直接本地编辑 |
| **远程网关** | `user@server:~/.openclaw/workspace` | SSH 到服务器编辑 |

### 连接到远程网关

```bash
# SSH 到远程服务器
ssh user@gateway-host

# 编辑工作区文件
vi ~/.openclaw/workspace/IDENTITY.md

# 或使用 scp 下载编辑后上传
scp user@gateway-host:~/.openclaw/workspace/IDENTITY.md ./
# 编辑后
scp IDENTITY.md user@gateway-host:~/.openclaw/workspace/
```

## 🛠️ 自定义引导模板

### 修改默认问题

如果要自定义引导问答流程，可以编辑种子模板：

```bash
# 编辑引导模板
vi ~/.openclaw/workspace/BOOTSTRAP.md
```

### 自定义示例

```markdown
# 🎯 智能体引导

欢迎！让我们来配置你的智能体。

## 必填信息

1. **智能体名称**：你希望如何称呼它？
2. **主要用途**：这个智能体用来做什么？
3. **专业领域**：它擅长什么领域？

## 可选信息

4. **个性风格**：严肃 / 幽默 / 友好？
5. **回复偏好**：简洁 / 详细？
6. **语言风格**：正式 / 随意？

---

> 提示：你可以随时在工作区目录手动编辑这些配置。
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 引导重复触发 | `BOOTSTRAP.md` 未删除 | 检查工作区文件 |
| 配置未生效 | 编辑了错误的工作区 | 确认网关主机位置 |
| 问答无法跳过 | 引导流程设计如此 | 完成回答或手动编辑 |

### 调试命令

```bash
# 检查工作区是否存在
ls -la ~/.openclaw/workspace/

# 查看引导状态
cat ~/.openclaw/workspace/BOOTSTRAP.md 2>/dev/null || echo "引导已完成"

# 查看生成的配置
cat ~/.openclaw/workspace/IDENTITY.md
cat ~/.openclaw/workspace/USER.md
cat ~/.openclaw/workspace/SOUL.md

# 远程网关检查
ssh user@gateway-host "ls -la ~/.openclaw/workspace/"
```

### 重新触发引导

如果需要重新配置智能体身份：

```bash
# 方法 1：删除引导完成的标记
rm ~/.openclaw/workspace/SOUL.md

# 方法 2：重新创建 BOOTSTRAP.md
echo "# 重启引导" > ~/.openclaw/workspace/BOOTSTRAP.md

# 方法 3：完全重置工作区（谨慎！）
# rm -rf ~/.openclaw/workspace
# 然后重新运行智能体
```

## 🎯 最佳实践

### 引导前准备

| 准备事项 | 说明 |
|----------|------|
| **明确用途** | 知道智能体用来做什么 |
| **定位清晰** | 是助手、专家还是伙伴 |
| **风格确定** | 希望什么样的回复风格 |
| **领域选择** | 专业领域或通用助手 |

### 身份配置建议

```markdown
# IDENTITY.md 示例（完整）

## 身份

你是一个专业的技术助手，名叫"小爪"。

## 角色

- 帮助用户解决技术问题
- 提供清晰的代码示例
- 解释复杂的技术概念

## 特点

- 回答简洁但完整
- 使用中文回复
- 需要时提供代码示例
- 不确定时主动询问

## 禁止

- 不要编造信息
- 不要执行危险操作
- 不要泄露敏感数据
```
