# 配置向导概述

OpenClaw 支持多种配置路径，您可以根据网关（Gateway）运行的位置和个人偏好选择最适合的方式。

## 💡 为什么需要配置向导？

配置向导是 OpenClaw 的**智能引导系统**，它解决了传统配置方式的痛点：

| 传统方式的问题 | 配置向导的解决方案 |
|--------------|------------------|
| 手动编辑复杂 JSON 配置文件 | 交互式问答，自动生成配置 |
| 不清楚哪些配置是必需的 | 智能检测，只问必要问题 |
| 配置错误难以排查 | 实时验证，立即发现问题 |
| 多步骤流程容易遗漏 | 自动引导，确保完整配置 |

### CLI 配置向导

**适用场景**：

- ✅ 希望完全控制每个配置项
- ✅ 需要在服务器或远程环境部署
- ✅ 熟悉命令行操作
- ✅ 需要 Windows (WSL2) 支持

**支持平台**：macOS、Linux、Windows (WSL2)

**启动方式**：

```bash
openclaw onboard
```

**优势**：

| 优势 | 说明 |
|------|------|
| **完全控制** | 精确配置网关、工作区、渠道和技能 |
| **远程友好** | 适合 SSH 远程部署 |
| **可脚本化** | 支持自动化集成 |
| **跨平台** | 统一的使用体验 |

**相关文档**：

- [CLI 配置向导详解](/start/wizard)
- [`openclaw onboard` 命令参考](/cli/onboard)

### macOS App 配置

**适用场景**：

- ✅ 使用 Mac 电脑（Apple Silicon 或 Intel）
- ✅ 偏好图形界面操作
- ✅ 希望获得最佳 macOS 集成体验

**启动方式**：

下载并打开 OpenClaw macOS App，首次运行会自动启动配置向导。

**优势**：

| 优势 | 说明 |

## 🔧 自定义提供商（Provider）

如果 OpenClaw 未内置您需要的 AI 提供商，但该提供商支持标准的 OpenAI 或 Anthropic API，您可以使用**自定义提供商**功能。

### 何时使用自定义提供商？

| 场景 | 示例 |
|------|------|
| 托管的兼容 API | Azure OpenAI、Google AI 代理 |
| 企业内部部署 | 公司内部的 AI 模型网关 |
| 第三方聚合服务 | 提供多模型统一接口的服务 |
| 开源模型部署 | Ollama、LocalAI 等本地部署 |

### 配置步骤

在 CLI 配置向导中选择 **Custom Provider（自定义提供商）**，系统将引导您完成：

#### 1. 选择 API 兼容性类型

| 兼容类型 | 说明 | 选择时机 |
|---------|------|----------|
| **OpenAI 兼容** | API 格式与 OpenAI 相同 | 使用 GPT 模型、Ollama 等 |
| **Anthropic 兼容** | API 格式与 Anthropic 相同 | 使用 Claude 系列模型 |
| **未知（自动检测）** | 让系统自动识别 | 不确定具体格式时 |

> **💡 专家提示**：如果您的提供商同时支持两种格式，优先选择 **Anthropic 兼容**，因为它支持流式响应和更丰富的功能。

#### 2. 配置连接信息

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **Base URL** | API 端点基础地址 | `https://api.example.com/v1` |
| **API Key** | 认证密钥（如需要） | `sk-xxxxx...` |
| **Model ID** | 要使用的模型标识符 | `gpt-4`、`claude-3-opus` |

#### 3. 设置模型别名（可选）

为模型指定一个易记的别名，方便后续引用：

```json
{
  "alias": "my-custom-model",
  "modelId": "gpt-4-turbo-preview"
}
```

#### 4. 选择端点 ID

如果需要配置多个自定义提供商，为每个端点指定唯一 ID：

```
示例配置：
- 端点 ID: azure-openai
- 端点 ID: ollama-local
- 端点 ID: company-gateway
```

### 配置示例

```bash
# 在 CLI 向导中选择 Custom Provider 后的交互示例

? 选择 API 兼容类型
  OpenAI 兼容
❯ Anthropic 兼容
  未知（自动检测）

? 输入 Base URL
https://api.example.com/v1

? 输入 API Key（可选）
sk-xxxxx...

? 输入 Model ID
claude-3-opus-20240229

? 输入模型别名（可选）
my-claude

? 输入端点 ID
custom-1
```

### 配置验证

配置完成后，系统会自动验证连接：

```bash
# 手动验证自定义提供商
openclaw health

# 输出示例：
# ✅ Gateway: healthy
# ✅ Auth: configured (custom-1)
# ✅ Custom Provider: connected
```

## ⚠️ 常见问题与解决方案

### 问题一：不确定应该选择哪种配置方式？

**决策树**：

```
您是 Mac 用户吗？
├── 是 → 偏好图形界面？ → 是 → macOS App 配置
│              └── 否 → CLI 配置
└── 否 → CLI 配置
```

### 问题二：自定义提供商连接失败

**排查步骤**：

| 检查项 | 命令/操作 |
|--------|----------|
| 1. 验证 Base URL | `curl -I https://your-api-url.com` |
| 2. 检查 API Key | 确认密钥格式和有效期 |
| 3. 查看详细日志 | `openclaw gateway --verbose` |
| 4. 测试模型 ID | 使用 `curl` 直接调用 API |

### 问题三：配置后模型不可用

**可能原因**：

1. **Model ID 错误**：确认模型标识符与提供商文档一致
2. **权限不足**：检查 API Key 是否有访问该模型的权限
3. **兼容性问题**：尝试切换 API 兼容类型
