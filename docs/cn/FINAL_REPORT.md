## 🎓 最佳实践总结

### 文档结构标准

**必备元素**（每篇文档）:
```markdown
---
version: "2026.2.17"          # ✅ 版本标注
last_updated: "2026-03-05"    # ✅ 更新时间
read_when:                    # ✅ 阅读时机
  - 场景 1
  - 场景 2
---

## 📋 前置条件              # ✅ 系统要求清单
- 检查项 1
- 检查项 2

## 核心内容

## ✅ 验证步骤              # ✅ 验证方法
- 验证命令 1
- 期望输出

## 💡 最佳实践            # ✅ 实用建议
- 建议 1
- 建议 2

## 🐛 常见问题              # ✅ FAQ
### 问题 1
**解决方案**: ...

## 📚 下一步                # ✅ 导航链接
- [相关文档 1](/link1)
- [相关文档 2](/link2)

## 📝 变更历史              # ✅ 变更记录
| 版本 | 日期 | 变更内容 |
|------|------|----------|
```

### 版本管理规范

**版本号格式**:
- 格式：`YYYY.M.D`（与产品版本一致）
- 示例：`2026.2.17`

**更新日期**:
- 格式：ISO 8601（`YYYY-MM-DD`）
- 位置：frontmatter 中的 `last_updated`

**变更历史**:
- 位置：文档末尾
- 格式：Markdown 表格
- 内容：版本号、日期、变更摘要

**更新触发**:
- 内容变更时必须更新 frontmatter
- 添加新章节时更新变更历史
- 重大更新时升级版本号

### 最佳实践编写指南

**内容要求**:
1. **可操作性**: 提供可直接复制的命令和配置
2. **场景化**: 针对具体使用场景提供建议
3. **对比性**: 使用表格对比推荐做法和不推荐做法
4. **示例化**: 提供完整的代码示例和配置模板
5. **量化**: 提供具体的参数建议和阈值

**组织结构**:
```markdown
## 💡 最佳实践

### 子主题
**简短说明**：一句话说明适用场景

**示例代码/命令**：
```bash
# 可直接复制的命令
```

**对比表格**（可选）：
| 做法 | 推荐 | 不推荐 |
|------|------|--------|
| 方面 | ✅ | ❌ |

**注意事项**：
- 关键点 1
- 关键点 2
```
