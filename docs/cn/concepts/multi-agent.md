## 💡 为什么需要多智能体？

### 类比：多智能体就像"多个独立的员工"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    多智能体员工类比                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：公司部门                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  财务部 ──► 独立办公室、独立文件柜、独立电脑                      │   │
│  │  技术部 ──► 独立办公室、独立文件柜、独立电脑                      │   │
│  │  人事部 ──► 独立办公室、独立文件柜、独立电脑                      │   │
│  │                                                                 │   │
│  │  特点：                                                          │   │
│  │  • 各部门完全隔离                                              │   │
│  │  • 有自己的资源和权限                                          │   │
│  │  • 通过前台路由访问                                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 场景：                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  智能体 A ──► 独立工作区、独立认证、独立会话                      │   │
│  │  智能体 B ──► 独立工作区、独立认证、独立会话                      │   │
│  │  智能体 C ──► 独立工作区、独立认证、独立会话                      │   │
│  │                                                                 │   │
│  │  特点：                                                          │   │
│  │  • 每个 agentId 完全隔离                                        │   │
│  │  • 有自己的人格（AGENTS.md/SOUL.md）                           │   │
│  │  • 通过 bindings 路由消息                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心价值

| 价值 | 说明 |
|------|------|
| **人格隔离** | 不同智能体有不同的性格和专长 |
| **数据隔离** | 会话和认证完全分开，互不干扰 |
| **账户隔离** | 多个 WhatsApp 号码共享一个服务器 |
| **权限隔离** | 每个智能体有不同的工具权限 |

## 📋 核心概念

### 什么是"一个智能体"？

一个**智能体**是一个完全独立作用域的大脑，拥有自己的：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       智能体隔离边界                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  智能体 ID: "home"                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  工作区（workspace）                                            │   │
│  │  ├── AGENTS.md（人设）                                          │   │
│  │  ├── SOUL.md（性格）                                            │   │
│  │  ├── USER.md（用户偏好）                                        │   │
│  │  └── skills/（智能体专用技能）                                   │   │
│  │                                                                 │   │
│  │  状态目录（agentDir）                                            │   │
│  │  ├── auth-profiles.json（认证配置）                             │   │
│  │  ├── models.json（模型注册表）                                  │   │
│  │  └── agent.json（智能体配置）                                   │   │
│  │                                                                 │   │
│  │  会话存储                                                        │   │
│  │  └── ~/.openclaw/agents/home/sessions/                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  ⚠️ 重要：主智能体凭证不会自动共享！                                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 术语对照

| 术语 | 说明 | 示例 |
|------|------|------|
| **agentId** | 一个"大脑"的唯一标识 | `"home"`, `"work"`, `"family"` |
| **accountId** | 渠道账户实例 | WhatsApp 的 `"personal"` vs `"biz"` |
| **binding** | 路由规则 | 将消息路由到指定智能体 |
| **workspace** | 工作区目录 | `~/.openclaw/workspace-<agentId>` |
| **agentDir** | 状态目录 | `~/.openclaw/agents/<agentId>/agent` |
| **mainKey** | 主会话键 | 智能体的默认会话标识 |

### 路径映射

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       目录结构                                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ~/.openclaw/                                                           │
│  ├── openclaw.json ──────────────► 全局配置                            │
│  │                                                                     │
│  ├── workspace ─────────────────► 主智能体工作区（main）               │
│  ├── workspace-home ────────────► 智能体 "home" 工作区                 │
│  ├── workspace-work ────────────► 智能体 "work" 工作区                  │
│  │                                                                     │
│  ├── agents/                                                           │
│  │   ├── main/                                                        │
│  │   │   ├── agent/ ────────────► 主智能体状态                        │
│  │   │   │   ├── auth-profiles.json                                   │
│  │   │   │   └── models.json                                          │

│  │   │                                                                 │
│  │   ├── home/                                                        │
│  │   │   ├── agent/ ────────────► 智能体 "home" 状态                  │

│  │       ├── agent/ ────────────► 智能体 "work" 状态                  │
│  │       └── sessions/ ─────────► 智能体 "work" 会话                  │
│  │                                                                     │
│  └── skills/ ──────────────────► 共享技能库                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🚀 快速开始

### 创建新智能体

```bash
# 启动向导创建新智能体
openclaw agents add work

# 查看所有智能体和绑定
openclaw agents list --bindings

# 验证智能体状态
openclaw agents status
```

### 单智能体模式（默认）

如果你什么都不做，OpenClaw 运行单个智能体：

| 属性 | 默认值 |
|------|--------|
| `agentId` | `"main"` |
| 会话键 | `agent:main:<mainKey>` |
| 工作区 | `~/.openclaw/workspace` |
| 状态目录 | `~/.openclaw/agents/main/agent` |

## 🔄 路由规则

### 绑定优先级

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       路由决策流程                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  收到入站消息                                                            │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  按优先级检查绑定                                                │   │
│  │                                                                 │   │
│  │  1. peer 精确匹配（私信/群组/频道 ID）                          │   │
│  │     例如：{ kind: "dm", id: "+15551234567" }                    │   │
│  │                                                                 │   │
│  │  2. guildId（Discord 服务器）                                   │   │
│  │                                                                 │   │
│  │  3. teamId（Slack 工作区）                                      │   │
│  │                                                                 │   │
│  │  4. accountId 精确匹配                                          │   │
│  │     例如：{ channel: "whatsapp", accountId: "personal" }       │   │
│  │                                                                 │   │
│  │  5. 渠道级匹配（accountId: "*"）                                │   │
│  │     例如：{ channel: "telegram" }                               │   │
│  │                                                                 │   │
│  │  6. 回退到默认智能体                                             │   │
│  │     • agents.list[].default=true 的智能体                       │   │
│  │     • 否则：第一个智能体                                         │   │
│  │     • 否则：main                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 路由示例

```
场景：WhatsApp 私信 + Telegram 群组

┌─────────────────────────────────────────────────────────────────────────┐
│  消息来源                   │  匹配绑定           │  智能体            │
├─────────────────────────────┼────────────────────┼─────────────────────┤
│  WhatsApp DM +15551230001   │  peer.id 精确匹配   │  alex              │
│  WhatsApp DM +15551230002   │  peer.id 精确匹配   │  mia               │
│  WhatsApp 群组              │  accountId: personal │  home              │
│  Telegram 私信              │  channel 级匹配     │  opus              │
│  Discord 服务器 X           │  guildId 匹配       │  discord-bot       │
│  未知来源                   │  回退默认           │  main              │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📱 配置场景

### 场景 1：两个 WhatsApp → 两个智能体

**目标**：个人和工作 WhatsApp 分别路由到不同智能体

```json5
{
  agents: {
    list: [
      {
        id: "home",
        default: true,
        name: "Home",
        workspace: "~/.openclaw/workspace-home",
        agentDir: "~/.openclaw/agents/home/agent",
      },
      {
        id: "work",
        name: "Work",
        workspace: "~/.openclaw/workspace-work",
        agentDir: "~/.openclaw/agents/work/agent",
      },
    ],
  },

  bindings: [
    { agentId: "home", match: { channel: "whatsapp", accountId: "personal" } },
    { agentId: "work", match: { channel: "whatsapp", accountId: "biz" } },
  ],

  channels: {
    whatsapp: {
      accounts: {
        personal: {
          authDir: "~/.openclaw/credentials/whatsapp/personal",
        },
        biz: {
          authDir: "~/.openclaw/credentials/whatsapp/biz",
        },
      },
    },
  },
}
```

### 场景 2：按渠道分割

**目标**：WhatsApp 用快速模型，Telegram 用强力模型

```json5
{
  agents: {
    list: [
      {
        id: "chat",
        name: "Everyday",
        workspace: "~/.openclaw/workspace-chat",
        model: "anthropic/claude-sonnet-4-5",
      },
      {
        id: "opus",
        name: "Deep Work",
        workspace: "~/.openclaw/workspace-opus",
        model: "anthropic/claude-opus-4-5",
      },
    ],
  },
  bindings: [
    { agentId: "chat", match: { channel: "whatsapp" } },
    { agentId: "opus", match: { channel: "telegram" } },
  ],
}
```

### 场景 3：特定用户路由到 Opus

**目标**：保持 WhatsApp 在快速智能体，但一个私信用 Opus

```json5
{
  agents: {
    list: [
      {
        id: "chat",
        name: "Everyday",
        workspace: "~/.openclaw/workspace-chat",
        model: "anthropic/claude-sonnet-4-5",
      },
      {
        id: "opus",
        name: "Deep Work",
        workspace: "~/.openclaw/workspace-opus",
        model: "anthropic/claude-opus-4-5",
      },
    ],
  },
  bindings: [
    // 对等方匹配优先级最高，放在前面
    { agentId: "opus", match: { channel: "whatsapp", peer: { kind: "dm", id: "+15551234567" } } },
    { agentId: "chat", match: { channel: "whatsapp" } },
  ],
}
```

### 场景 4：家庭群组智能体

**目标**：专用智能体绑定到单个 WhatsApp 群组

```json5
{
  agents: {
    list: [
      {
        id: "family",
        name: "Family",
        workspace: "~/.openclaw/workspace-family",
        identity: { name: "Family Bot" },
        groupChat: {
          mentionPatterns: ["@family", "@familybot", "@Family Bot"],
        },
        sandbox: {
          mode: "all",
          scope: "agent",
        },
        tools: {
          allow: [
            "exec",
            "read",
            "sessions_list",
            "sessions_history",
            "sessions_send",
            "sessions_spawn",
            "session_status",
          ],
          deny: ["write", "edit", "apply_patch", "browser", "canvas", "nodes", "cron"],
        },
      },
    ],
  },
  bindings: [
    {
      agentId: "family",
      match: {
        channel: "whatsapp",
        peer: { kind: "group", id: "120363999999999999@g.us" },
      },
    },
  ],
}
```

### 场景 5：多用户共享一个 WhatsApp

**目标**：不同私聊路由到不同智能体

```json5
{
  agents: {
    list: [
      { id: "alex", workspace: "~/.openclaw/workspace-alex" },
      { id: "mia", workspace: "~/.openclaw/workspace-mia" },
    ],
  },
  bindings: [
    { agentId: "alex", match: { channel: "whatsapp", peer: { kind: "dm", id: "+15551230001" } } },
    { agentId: "mia", match: { channel: "whatsapp", peer: { kind: "dm", id: "+15551230002" } } },
  ],
  channels: {
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15551230001", "+15551230002"],
    },
  },
}
```

## 🛡️ 每智能体沙箱和工具

### 配置示例

```json5
{
  agents: {
    list: [
      {
        id: "personal",
        workspace: "~/.openclaw/workspace-personal",
        sandbox: {
          mode: "off",  // 个人智能体无沙箱
        },
        // 无工具限制 - 所有工具可用
      },
      {
        id: "family",
        workspace: "~/.openclaw/workspace-family",
        sandbox: {
          mode: "all",     // 始终沙箱隔离
          scope: "agent",  // 每智能体一个容器
          docker: {
            setupCommand: "apt-get update && apt-get install -y git curl",
          },
        },
        tools: {
          allow: ["read"],                    // 仅 read 工具
          deny: ["exec", "write", "edit", "apply_patch"],
        },
      },
    ],
  },
}
```

### 隔离好处

| 好处 | 说明 |
|------|------|
| **安全隔离** | 限制不受信任智能体的工具 |
| **资源控制** | 沙箱隔离特定智能体 |
| **灵活策略** | 每智能体不同权限 |

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 消息路由错误 | 绑定顺序错误 | 精确匹配放在前面 |
| 会话混淆 | 共享 agentDir | 使用独立 agentDir |
| 工具不可用 | 沙箱限制 | 检查 tools.allow/deny |
| 认证失败 | auth-profiles 未复制 | 复制认证文件 |

### 调试技巧

```bash
# 查看所有智能体和绑定
openclaw agents list --bindings
```

## 🎯 最佳实践

### 设计原则

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    多智能体设计原则                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 明确隔离边界                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 每个智能体有明确的用途                                        │   │
│  │  • 不要为了"分离"而创建不必要的智能体                            │   │
│  │  • 相关功能可以放在同一智能体中                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 绑定顺序很重要                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 精确匹配（peer.id）放在最前面                                │   │
│  │  • accountId 匹配在中间                                         │   │
│  │  • 渠道级匹配（fallback）放在最后                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. 合理使用沙箱                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 不受信任的场景（群组、公共渠道）启用沙箱                      │   │
│  │  • 个人使用可以关闭沙箱以提高性能                                │   │
│  │  • 使用 tools.allow 明确限制权限                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 命名规范                                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • agentId 使用小写和连字符：work, personal, family-bot        │   │
│  │  • accountId 使用描述性名称：personal, biz, team-alpha          │   │
│  │  • 保持命名一致，便于维护                                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 场景推荐

| 场景 | 推荐配置 |
|------|----------|
| **个人使用** | 单智能体，无沙箱 |
| **家庭共享** | 多智能体，按用户绑定 |
| **团队协作** | 多智能体，按群组绑定 |
| **生产环境** | 沙箱隔离，严格工具限制 |
