## 💡 为什么需要时区处理？

### 类比：时区处理就像"世界时钟"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    世界时钟类比                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：跨国团队协作                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  纽约（EST）───► 下午 3 点 ──► 太平洋时间（PST）：中午 12 点     │   │
│  │  伦敦（GMT）───► 下午 8 点 ──► 东京（JST）：晚上 11 点          │   │
│  │                                                                 │   │
│  │  问题：如果没有统一参考，时间会混乱                            │   │
│  │  解决：使用 UTC 作为"通用语言"                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 场景：AI 聊天时间显示                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户 A（北京） ──► "昨天 8 点"                                │   │
│  │  用户 B（纽约） ──► "昨天 7 点"                                │   │
│  │                                                                 │   │
│  │  AI 困惑：到底是哪个时间？                                       │   │
│  │  解决：统一格式化时间戳                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心原则

| 原则 | 说明 |
|------|------|
| **单一参考时间** | 模型看到一致的时间格式 |
| **本地化显示** | 用户看到自己熟悉的时区 |
| **原始数据保留** | 保留提供商的原始时间戳 |

## 📨 消息信封时间

### 信封格式

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       消息信封结构                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  默认信封格式：                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [Provider Sender Date Timezone] message text                  │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  [Telegram Alice 2026-01-05 16:26 PST] 你好！                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  时间戳默认为主机本地时间，精确到分钟                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 配置选项

```json5
{
  agents: {
    defaults: {
      // 时区设置
      envelopeTimezone: "local",    // "utc" | "local" | "user" | IANA 时区

      // 时间戳显示
      envelopeTimestamp: "on",    // "on" | "off"

      // 已用时间显示
      envelopeElapsed: "on",       // "on" | "off"

      // 用户时区（用于 "user" 模式）
      userTimezone: "America/New_York"
    }
  }
}
```

### 时区模式

| 模式 | 说明 | 示例输出 |
|------|------|----------|
| **local** | 主机本地时间 | `2026-01-05 16:26 PST` |
| **utc** | UTC 时间 | `2026-01-06 00:26 UTC` |
| **user** | 用户配置时区 | `2026-01-05 19:26 EST` |
| **IANA 时区** | 固定时区 | `2026-01-06 01:26 Europe/Vienna` |

### 显示示例

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       不同配置的输出示例                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  本地时间（默认）：                                                     │
│  [Signal Alice +1555 2026-01-18 00:19 PST] hello                        │
│                                                                         │
│  固定时区（欧洲/维也纳）：                                              │
│  [Signal Alice +1555 2026-01-18 09:19 Europe/Vienna] hello             │
│                                                                         │
│  UTC 时间：                                                             │
│  [Signal Alice +1555 2026-01-18T08:19Z] hello                          │
│                                                                         │
│  带已用时间：                                                           │
│  [Signal Alice +1555 +2m 2026-01-18T05:19Z] follow-up                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🛠️ 工具调用时间

### 时间字段

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    工具调用返回的时间字段                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  工具调用返回（例如 channels.discord.readMessages）：                    │
│  {                                                                      │
│    messages: [                                                         │
│      {                                                                 │
│        // 原始提供商字段（保持不变）                                   │
│        timestamp: "2026-01-18T08:19:00.000Z",                          │
│                                                                         │
│        // OpenClaw 规范化字段（新增）                                   │
│        timestampMs: 1737171940000,        # UTC 纪元毫秒数               │
│        timestampUtc: "2026-01-18T08:19:00Z"  # ISO 8601 UTC 字符串        │
│      }                                                                 │
│    ]                                                                   │
│  }                                                                      │
│                                                                         │
│  优势：                                                                  │
│  • 提供统一的 UTC 参考点                                             │
│  • 保留原始数据用于调试                                             │
│  • 支持跨时区应用                                                   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `timestampMs` | number | UTC 纪元毫秒数 |
| `timestampUtc` | string | ISO 8601 UTC 字符串 |
| 原始字段 | - | 提供商原始格式（保留） |

## 🕐 系统提示词时间

### 用户时区配置

```json5
{
  agents: {
    defaults: {
      // 设置用户的本地时区
      userTimezone: "Asia/Shanghai",

      // 或使用其他 IANA 时区
      // userTimezone: "America/New_York"
      // userTimezone: "Europe/London"
      // userTimezone: "UTC"

      // 时间格式
      timeFormat: "auto"  // "auto" | "12" | "24"
    }
  }
}
```

### 系统提示词中的时间显示

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    系统提示词中的时间部分                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Current Date & Time 部分：                                           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Current Date & Time                                            │   │
│  │  • 2026年1月18日 星期六                                        │   │
│  │  • 上午 8:30 (Asia/Shanghai, UTC+8)                              │   │
│  │                                                                 │   │
│  │  Time format: 12-hour                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  时间格式选项：                                                          │
│  • auto：根据地区自动选择 12/24 小时制                               │
│  • 12：强制 12 小时制（上午/下午）                                   │
│  • 24：强制 24 小时制（0-23）                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 自动时区解析

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    时区自动解析                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  如果未设置 userTimezone：                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  OpenClaw 在运行时自动解析主机时区                               │   │
│  │                                                                 │   │
│  │  • macOS: 从系统偏好设置读取                                     │   │
│  │  • Linux: 从 /etc/localtime 读取                                   │   │
│  │  • Windows: 从系统设置读取                                       │   │
│  │                                                                 │   │
│  │  优点：无需手动配置                                             │   │
│  │  注意：主机时区可能不是用户时区（远程服务器）                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 时间显示错误 | `envelopeTimezone` 配置错误 | 检查时区配置 |
| 时间格式不对 | `timeFormat` 设置问题 | 尝试 "auto" 模式 |
| 已用时间显示 | `envelopeElapsed` 开启 | 设置为 "off" 关闭 |
| 系统提示词时间错误 | `userTimezone` 未配置 | 设置用户时区 |

### 调试命令

```bash
# 查看当前时区配置
openclaw config get agents.defaults

# 查看主机时区
# Linux/Mac
date +%Z
timedatectl  # Linux

# 查看系统提示词
/context detail
```

## 🎯 最佳实践

### 配置建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    时区配置建议                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 本地部署（个人使用）                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  envelopeTimezone: "local"                                       │   │
│  │  userTimezone: 不设置（自动解析主机时区）                        │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 显示本地时间，易于理解                                       │   │
│  │  • 无需手动配置                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 服务器部署（多时区用户）                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  envelopeTimezone: "utc"                                        │   │
│  │  userTimezone: "user"  # 使用每个用户配置的时区                    │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 统一参考时间（UTC）                                           │   │
│  │  • 每个用户看到自己的时区                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. 固定时区需求                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  envelopeTimezone: "Asia/Shanghai"                              │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 所有用户看到统一时区                                         │   │
│  │  • 适合团队协作场景                                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 跨时区用户配置

```json5
{
  agents: {
    defaults: {
      // 使用用户各自配置的时区
      envelopeTimezone: "user",
      timeFormat: "auto"
    },
    list: [
      {
        id: "user-ny",
        userTimezone: "America/New_York"
      },
      {
        id: "user-london",
        userTimezone: "Europe/London"
      },
      {
        id: "user-shanghai",
        userTimezone: "Asia/Shanghai"
      }
    ]
  }
}
```
