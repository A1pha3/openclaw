## 💡 为什么需要故障转移？

### 类比：故障转移就像"备用卡机制"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    备用卡机制类比                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：信用卡支付                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  场景：网上购物                                                  │   │
│  │                                                                 │   │
│  │  主卡刷卡 ──► 拒绝（余额不足）                                  │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  备用卡 1 ──► 拒绝（超过限额）                                  │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  备用卡 2 ──► 成功 ✓                                            │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 一张卡失败不影响支付                                        │   │
│  │  • 自动尝试下一张卡                                             │   │
│  │  • 提高成功率                                                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 场景：AI 模型调用                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  主认证配置文件 ──► 失败（速率限制）                            │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  备用配置文件 1 ──► 失败（配额耗尽）                             │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  备用配置文件 2 ──► 成功 ✓                                      │   │
│  │                                                                 │   │
│  │  所有配置文件失败？                                              │   │
│  │     ▼                                                           │   │
│  │  回退到备用模型 ──► 尝试其他提供商                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 两阶段故障转移

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    两阶段故障转移流程                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  AI 调用失败                                                            │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  阶段 1：认证配置文件轮换                                       │   │
│  │                                                                 │   │
│  │  同一提供商内尝试不同的认证配置文件：                            │   │
│  │  配置文件 A ──► 失败（速率限制）                                │   │
│  │  配置文件 B ──► 失败（配额耗尽）                                │   │
│  │  配置文件 C ──► 成功 ✓                                         │   │
│  │                                                                 │   │
│  │  所有配置文件都失败？→ 进入阶段 2                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  阶段 2：模型回退                                             │   │
│  │                                                                 │   │
│  │  尝试备用模型（不同提供商）：                                  │   │
│  │  主模型 ──► 所有配置文件失败                                    │   │
│  │  备用模型 1 ──► 尝试其配置文件                                  │   │
│  │  备用模型 2 ──► 尝试其配置文件                                  │   │
│  │                                                                 │   │
│  │  成功 ✓ 或最终失败                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🔄 认证配置文件轮换

### 轮换顺序

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    配置文件选择顺序                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 显式配置（优先级最高）                                             │
│     auth.order[provider] = ["profile1", "profile2"]                   │
│                                                                         │
│  2. 已配置的配置文件                                                   │
│     auth.profiles 中按提供商过滤的条目                                  │
│                                                                         │
│  3. 已存储的配置文件                                                   │
│     auth-profiles.json 中该提供商的所有条目                             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 自动轮询顺序

当没有显式配置时，使用轮询顺序：

| 优先级 | 主键 | 次键 |
|--------|------|------|
| 1 | OAuth（优先） | 最旧的 `lastUsed` |
| 2 | API 密钥 | 最旧的 `lastUsed` |
| 3 | 冷却/禁用的配置文件 | 最早过期时间 |

### 会话粘性

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    会话粘性机制                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  OpenClaw 为每个会话固定认证配置文件                                    │
│                                                                         │
│  选择配置文件 ──► 固定到会话 ──► 重用直到：                            │
│     │                                  │                               │
│     │                                  ├─ 会话重置（/new）           │
│     │                                  ├─ 压缩完成                  │
│     │                                  └─ 配置文件冷却              │
│                                                                         │
│  好处：                                                                  │
│  • 保持提供商缓存热度                                                  │
│  • 避免频繁切换配置文件                                                │
│  • 提高性能                                                            │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### OAuth"看起来丢失"问题

**症状**：消息之间切换使用 OAuth 和 API 密钥

**原因**：两个配置文件类型共存，自动轮换

**解决方案**：

```json5
{
  auth: {
    order: {
      anthropic: ["anthropic:my-oauth-profile"]  // 固定使用 OAuth
    }
  }
}
```

## ❄️ 冷却时间

### 冷却机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    冷却时间指数退避                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  失败次数              冷却时间                                          │
│     │                    │                                             │
│     ├─► 第 1 次失败 ────► 1 分钟                                      │
│     │                    │                                             │
│     ├─► 第 2 次失败 ────► 5 分钟                                      │
│     │                    │                                             │
│     ├─► 第 3 次失败 ────► 25 分钟                                     │
│     │                    │                                             │
│     └─► 第 4+ 次失败 ───► 1 小时（上限）                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 触发冷却的错误

| 错误类型 | 说明 | 是否冷却 |
|---------|------|----------|
| 认证失败 | 无效 API 密钥 | ✅ 是 |
| 速率限制 | 429 错误 | ✅ 是 |
| 超时 | 看起来像速率限制 | ✅ 是 |
| 格式错误 | 值得故障转移 | ✅ 是 |
| 其他错误 | 通常不推进回退 | ❌ 否 |

### 冷却状态存储

```json
{
  "usageStats": {
    "anthropic:profile1": {
      "lastUsed": 1736160000000,
      "cooldownUntil": 1736160600000,
      "errorCount": 2
    }
  }
}
```

### 回退流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    模型回退流程                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  配置：                                                                  │
│  {                                                                      │
│    agents: {                                                            │
│      defaults: {                                                        │
│        model: {                                                         │
│          primary: "anthropic/claude-opus-4-5",                         │
│          fallbacks: [                                                   │
│            "anthropic/claude-sonnet-4-5",                              │
│            "openai/gpt-4o",                                            │
│            "google/gemini-2.5-pro"                                    │
│          ]                                                              │
│        }                                                                │
│      }                                                                  │
│    }                                                                    │
│  }                                                                      │
│                                                                         │
│  执行流程：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  claude-opus-4-5 ──► 所有配置文件失败                             │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  claude-sonnet-4-5 ──► 所有配置文件失败                           │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  gpt-4o ──► 所有配置文件失败                                     │   │
│  │     │                                                           │   │
│  │     ▼                                                           │   │
│  │  gemini-2.5-pro ──► 成功 ✓ 或最终失败                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 推进回退的错误

| 错误类型 | 是否推进回退 | 说明 |
|---------|-------------|------|
| 认证失败 | ✅ 是 | 所有配置文件都失败 |
| 速率限制 | ✅ 是 | 所有配置文件都冷却 |
| 超时 | ✅ 是 | 所有配置文件都失败 |
| 格式错误 | ❌ 否 | 不会因格式错误切换模型 |

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 频繁切换配置文件 | 没有设置 `auth.order` | 固定配置文件顺序 |
| 配置文件不使用 | 在冷却中 | 等待冷却时间结束 |
| 模型不回退 | `fallbacks` 未配置 | 添加备用模型列表 |
| OAuth"丢失" | 自动轮换 | 使用 `auth.order` 固定 |

### 调试命令

```bash
# 查看配置文件状态
openclaw models status --verbose

# 检查冷却状态
openclaw models status --json | jq '.auth.profiles'

# 查看模型配置
openclaw config get agents.defaults.model

# 测试特定配置文件
/model claude-opus@anthropic:my-profile
```

## 🎯 最佳实践

### 配置建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    故障转移配置建议                                     │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 设置多个认证配置文件                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 至少 2-3 个配置文件                                           │   │
│  │  • 使用不同的账户/密钥                                          │   │
│  │  • 混合 OAuth 和 API 密钥                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 配置备用模型                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 至少 2 个备用模型                                             │   │
│  │  • 使用不同提供商（降低单点故障）                               │   │
│  │  • 考虑性能和成本平衡                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. 固定配置文件顺序（如需要）                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  auth.order: {                                                   │   │
│  │    anthropic: ["anthropic:primary", "anthropic:backup"]         │   │
│  │  }                                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 监控状态                                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 定期检查冷却状态                                             │   │
│  │  • 监控账单余额                                                 │   │
│  │  • 及时更新过期密钥                                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 示例配置

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-5",
        fallbacks: [
          "anthropic/claude-sonnet-4-5",    // 同提供商，较便宜
          "openai/gpt-4o",                   // 不同提供商
          "google/gemini-2.5-pro",          // 另一个备选
        ]
      }
    }
  },
  auth: {
    // 固定配置文件顺序（可选）
    order: {
      anthropic: ["anthropic:work", "anthropic:personal"]
    },
    // 账单退避配置
    cooldowns: {
      billingMaxHours: 24,
      failureWindowHours: 24
    }
  }
}
```
