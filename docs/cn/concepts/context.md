## 💡 什么是上下文？

### 类比：上下文就像"工作记忆"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    上下文类比                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  人类工作记忆：                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  当前对话中的信息（工作记忆）                                    │   │
│  │  • 对方刚才说了什么                                             │   │
│  │  • 我们正在讨论什么主题                                         │   │
│  │  • 相关的背景知识                                               │   │
│  │                                                                 │   │
│  │  限制：容量有限，需要不断刷新                                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  AI 模型上下文：                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  每次运行发送给模型的所有内容                                    │   │
│  │  • 系统提示词（规则、工具、Skills）                             │   │
│  │  • 对话历史（用户和 AI 的消息）                                 │   │
│  │  • 工具调用和结果                                               │   │
│  │  • 图片、文件等附件                                             │   │
│  │                                                                 │   │
│  │  限制：上下文窗口（Token 数量限制）                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  上下文 vs 记忆：                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  上下文                              记忆                        │   │
│  │  • 当前窗口内的内容                  • 磁盘存储的文件            │   │
│  │  • 每次运行重新构建                  • 持久保存                  │   │
│  │  • 受 Token 限制                    • 可以重新加载              │   │
│  │  • 模型当前"看到"的                 • 长期知识库                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **上下文** | 每次运行发送给模型的所有内容 |
| **上下文窗口** | 模型可处理的最大 Token 数 |
| **记忆** | 存储在磁盘上的持久化文件 |
| **压缩** | 将旧对话总结为摘要释放窗口 |
| **修剪** | 临时清理旧工具结果 |

## 🧱 上下文构成

### 完整构成图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    上下文构成                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  系统提示词（固定）                                              │   │
│  │  ├─ Tooling ──────────► 工具列表 + 描述                         │   │
│  │  ├─ Skills ───────────► 可用 Skills 列表                        │   │
│  │  ├─ Safety ───────────► 安全防护提醒                             │   │
│  │  ├─ Workspace ─────────► 工作区路径                             │   │
│  │  ├─ Runtime ───────────► 主机/OS/Node/模型                      │   │
│  │  └─ Project Context ───► 注入的引导文件                         │   │
│  │      ├─ AGENTS.md                                              │   │
│  │      ├─ SOUL.md                                                │   │
│  │      ├─ TOOLS.md                                               │   │
│  │      ├─ IDENTITY.md                                            │   │
│  │      ├─ USER.md                                                │   │
│  │      └─ HEARTBEAT.md                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              +                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  对话历史（动态）                                                │   │
│  │  ├─ 用户消息                                                   │   │
│  │  ├─ AI 回复                                                   │   │
│  │  └─ 压缩摘要（如有）                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              +                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  工具调用和结果（动态）                                          │   │
│  │  ├─ 工具调用参数                                               │   │
│  │  ├─ 命令输出                                                   │   │
│  │  ├─ 文件内容                                                   │   │
│  │  └─ 图片/音频转录                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              =                                         │
│                              ▼                                         │
│                         总 Token 数                                   │
│                                                                         │
│  限制：模型的上下文窗口（如 200K tokens）                               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### Token 估算

```
字符 ≈ Token × 4

示例转换：
- 1,000 字符 ≈ 250 token
- 10,000 字符 ≈ 2,500 token
- 100,000 字符 ≈ 25,000 token
- 200,000 字符 ≈ 50,000 token
```

## 🔍 检查上下文

### 快速命令

| 命令 | 用途 | 输出 |
|------|------|------|
| `/status` | 窗口使用情况 | 使用百分比、会话设置 |
| `/context list` | 注入内容概览 | 文件列表、大小 |
| `/context detail` | 详细分解 | 每项的精确 Token |

## 📊 上下文开销分析

### 主要来源

| 来源 | 典型 Token | 可优化 |
|------|-----------|--------|
| 系统提示词 | 5K-15K | ⭐⭐⭐ |
| 工具 schema | 5K-20K | ⭐⭐ |
| 引导文件 | 2K-10K | ⭐⭐⭐⭐ |
| Skills 列表 | 500-2K | ⭐ |
| 对话历史 | 变化大 | ⭐⭐⭐⭐⭐ |
| 工具结果 | 变化大 | ⭐⭐⭐⭐ |

### 优化策略

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    上下文优化策略                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  高优先级（显著效果）：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 压缩长对话 ──► /compact                                      │   │
│  │  • 精简引导文件 ──► 删除冗余内容                               │   │
│  │  • 限制工具结果 ──► 启用修剪                                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  中优先级（适度效果）：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 禁用不需要的工具 ──► 工具策略                                │   │
│  │  • 减少 Skills 数量 ──► 按需安装                                │   │
│  │  • 降低 bootstrapMaxChars ──► 减少注入                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  低优先级（微小效果）：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 优化系统提示词 ──► 通常已优化                                │   │
│  │  • 压缩工具 schema ──► 由工具定义                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 引导文件注入

| 文件 | 状态 | 说明 |
|------|------|------|
| `OK` | 正常 | 完整注入 |
| `TRUNCATED` | 截断 | 超过 `bootstrapMaxChars` |
| `MISSING` | 缺失 | 文件不存在 |

## 🔧 工具 Schema 成本

### 双重成本

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    工具成本分析                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  工具以两种方式计入上下文：                                              │
│                                                                         │
│  1. 工具列表文本（可见）                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  Tooling:                                                      │   │
│  │  • read - 读取文件内容                                         │   │
│  │  • write - 写入文件内容                                        │   │
│  │  • exec - 执行命令                                             │   │
│  │                                                                 │   │
│  │  约 1,000 字符 / 250 token                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 工具 schema（JSON，不可见但计入）                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  {                                                             │   │
│  │    "name": "browser",                                         │   │
│  │    "inputSchema": { ... }  ← 可能有几千字符                   │   │
│  │  }                                                             │   │
│  │                                                                 │   │
│  │  约 10,000 字符 / 2,500 token                                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  总成本：工具列表 + 工具 schema = 每个工具可能 3K+ token                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🎯 优化配置

### 减少引导文件大小

```json5
{
  agents: {
    defaults: {
      bootstrapMaxChars: 10000  // 默认 20000
    }
  }
}
```

### 工具策略

```json5
{
  agents: {
    defaults: {
      tools: {
        allow: ["read", "write", "exec"],  // 仅允许这些
        deny: ["browser", "multi-modal"]   // 拒绝这些
      }
    }
  }
}
```

### 会话修剪

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "5m"
      }
    }
  }
}
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 上下文超限 | 历史太长 | 运行 `/compact` |
| 响应变慢 | 工具 schema 大 | 禁用不需要的工具 |
| 引导文件未注入 | 文件缺失 | 检查工作区路径 |
| Token 消耗高 | 大量工具结果 | 启用修剪 |

### 调试技巧

```bash
# 1. 查看完整上下文分解
/context detail

# 2. 检查压缩历史
/compact --dry-run

# 3. 监控每次使用
/usage tokens

# 4. 开始新会话（如果无法压缩）
/new
```
