## 💡 什么是系统提示词？

### 类比：系统提示词就像"工作手册"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    系统提示词类比                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  新员工入职：                                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  公司手册                                                        │   │
│  │  ├── 公司规则和价值观                                          │   │
│  │  ├── 工作流程和标准                                            │   │
│  │  ├── 可用的工具和资源                                          │   │
│  │  ├── 联系方式和沟通规范                                        │   │
│  │  └── 特殊情况处理                                              │   │
│  │                                                                 │   │
│  │  员工阅读手册后知道：                                           │   │
│  │  • 应该做什么，不应该做什么                                     │   │
│  │  • 遇到问题找谁帮忙                                             │   │
│  │  • 如何使用公司工具                                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 智能体：                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  系统提示词                                                      │   │
│  │  ├── 可用工具和用法                                             │   │
│  │  ├── 安全防护提醒                                               │   │
│  │  ├── Skills 加载方式                                            │   │
│  │  ├── 工作区路径和文档                                           │   │
│  │  ├── 当前时间和时区                                             │   │
│  │  ├── 运行时环境信息                                             │   │
│  │  └── 用户偏好和习惯（引导文件）                                 │   │
│  │                                                                 │   │
│  │  智能体读取提示词后知道：                                         │   │
│  │  • 可以使用哪些工具                                             │   │
│  │  • 如何处理用户请求                                             │   │
│  │  • 在哪里找到文档帮助                                           │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心特点

| 特点 | 说明 |
|------|------|
| **OpenClaw 拥有** | 不使用 pi-coding-agent 默认提示词 |
| **动态组装** | 每次运行时重新构建 |
| **模块化** | 固定部分 + 可选引导文件 |
| **缓存优化** | 保持稳定部分以提升缓存效率 |

```
│  │  9. Reply Tags ───────────► 回复标签语法                        │   │
│  │  10. Heartbeats ──────────► 心跳提示和确认                      │   │
│  │  11. Runtime ─────────────► 主机/OS/Node/模型                   │   │

│  └─────────────────────────────────────────────────────────────────┘   │
│                              +                                         │
│                              ▼                                         │
│  引导文件注入（工作区文件）                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  ┌─────────────────────────────────────────────────────────┐   │   │
│  │  │ Project Context:                                         │   │   │
│  │  │ • AGENTS.md ─────────► 操作指南 + 记忆                    │   │   │
│  │  │ • SOUL.md ───────────► 人设和风格                         │   │   │
│  │  │ • TOOLS.md ──────────► 工具使用说明                       │   │   │
│  │  │ • IDENTITY.md ───────► 智能体身份                         │   │   │
│  │  │ • USER.md ───────────► 用户信息                           │   │   │
│  │  │ • HEARTBEAT.md ──────► 心跳检查清单                       │   │   │
│  │  │ • MEMORY.md ─────────► 长期记忆（可选）                    │   │   │
│  │  └─────────────────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                              =                                         │
│                              ▼                                         │
│                         最终系统提示词                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 部分说明

| 部分 | 内容 | 用途 |
|------|------|------|
| **Tooling** | 当前工具列表 + 描述 | 告诉 AI 可用工具 |
| **Safety** | 防护提醒 | 避免危险行为 |
| **Skills** | 可用 Skills 列表 | 按需加载 Skill |
| **Workspace** | 工作目录路径 | 文件操作基准 |
| **Sandbox** | 隔离信息 | 安全边界 |
| **Date & Time** | 时区 | 本地化时间 |
| **Runtime** | 环境信息 | 调试和日志 |

## 📁 工作区引导注入

### 引导文件映射

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    引导文件注入                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ~/.openclaw/workspace/                                                 │
│  ├── AGENTS.md ──────► 操作指南和记忆                                   │
│  ├── SOUL.md ────────► 人设、边界、语气                                 │
│  ├── TOOLS.md ───────► 工具使用说明                                     │
│  ├── IDENTITY.md ────► 智能体名称/风格/表情                             │
│  ├── USER.md ────────► 用户档案 + 偏好                                 │
│  ├── HEARTBEAT.md ───► 心跳运行检查清单                                 │
│  └── BOOTSTRAP.md ──► 一次性首次运行（仅新工作区）                      │
│                                                                         │
│  注入规则：                                                              │
│  • 大文件会被截断（默认 20000 字符）                                   │
│  • 缺失文件注入"文件缺失"标记                                          │
│  • 可以通过钩子拦截和修改                                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 注入行为

| 文件状态 | 注入内容 |
|----------|----------|
| 文件存在且 ≤ 限制 | 完整内容 |
| 文件存在且 > 限制 | 截断内容 + 标记 |
| 文件不存在 | 缺失文件标记 |
| 空文件 | 跳过注入 |

### 配置限制

```json5
{
  agents: {
    defaults: {
      bootstrapMaxChars: 20000  // 每个文件的最大字符数
    }
  }
}
```

### 钩子拦截

通过 `agent:bootstrap` 钩子可以修改注入：

```javascript
// 修改或替换引导文件
agent:bootstrap: {
  files: {
    "SOUL.md": "自定义角色内容..."
  }
}
```

### 检查注入大小

```bash
# 列出注入文件的大小
/context list

# 详细查看上下文使用
/context detail
```

## 🔧 提示词模式

### 模式对比

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    提示词模式对比                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  full（默认）───────────────────────────────────────────────            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  包含所有部分：                                                 │   │
│  │  ✓ Tooling、Safety、Workspace、Sandbox、Date/Time               │   │
│  │  ✓ Skills、Memory Recall、Self-Update                          │   │
│  │  ✓ User Identity、Reply Tags、Messaging                        │   │
│  │  ✓ Silent Replies、Heartbeats、Runtime、Reasoning              │   │
│  │                                                                 │   │
│  │  适用：主要智能体                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  minimal ────────────────────────────────────────────────────           │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  精简部分：                                                     │   │
│  │  ✓ Tooling、Safety、Workspace、Sandbox、Date/Time、Runtime      │   │
│  │  ✗ Skills、Memory Recall、Self-Update、User Identity...        │   │
│  │                                                                 │   │
│  │  适用：子智能体                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  none ────────────────────────────────────────────────────────          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  仅基本身份行                                                   │   │
│  │                                                                 │   │
│  │  适用：特殊场景                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 模式选择

| 模式 | 用途 | 特点 |
|------|------|------|
| `full` | 主要智能体 | 完整功能，最大上下文 |
| `minimal` | 子智能体 | 精简提示词，专注任务 |
| `none` | 特殊场景 | 最小上下文，快速响应 |

## 🛠️ Skills 注入

### Skills 列表格式

```xml
<available_skills>
  <skill>
    <name>commit</name>
    <description>创建 Git 提交</description>
    <location>workspace:/skills/commit</location>
  </skill>
  <skill>
    <name>review-pr</name>
    <description>审查 Pull Request</description>
    <location>builtin:/skills/review-pr</location>
  </skill>
</available_skills>
```

### 位置类型

| 位置前缀 | 说明 | 示例 |
|----------|------|------|
| `workspace:` | 工作区 Skills | `workspace:/skills/commit` |
| `hosted:` | 托管 Skills | `hosted:/skills/code-review` |
| `builtin:` | 内置 Skills | `builtin:/skills/review-pr` |

### 按需加载

```
提示词指示模型：
1. 查看可用 Skills 列表
2. 根据需要使用 read 工具加载 SKILL.md
3. 按 Skill 指令执行任务

优势：
• 保持基础提示词小巧
• 支持有针对性的 Skill 使用
• 减少 Token 消耗
```

## 📅 时间处理

### 时区配置

```json5
{
  agents: {
    defaults: {
      userTimezone: "Asia/Shanghai",
      timeFormat: "auto"  // auto | 12 | 24
    }
  }
}
```

### 提示词内容

为保持缓存稳定，系统提示词现在只包含：

```
Current Date & Time:
Timezone: Asia/Shanghai
```

### 获取当前时间

当智能体需要当前时间时：

```bash
# 使用 session_status 工具
session_status()
# 返回包含时间戳的状态卡片
```

### 格式选项

| 格式 | 说明 | 示例 |
|------|------|------|
| `auto` | 根据地区自动选择 | 中国用 24 小时 |
| `12` | 12 小时制 | 3:30 PM |
| `24` | 24 小时制 | 15:30 |

## 🎯 配置示例

### 基础配置

```json5
{
  agents: {
    defaults: {
      userTimezone: "Asia/Shanghai",
      timeFormat: "24",
      bootstrapMaxChars: 20000
    }
  }
}
```

### 自定义引导文件

```json5
{
  agents: {
    defaults: {
      workspace: "~/.openclaw/workspace"
    }
  }
}
```

然后在工作区创建文件：

```markdown
# ~/.openclaw/workspace/SOUL.md

# 角色设定

你是一个专业但友好的技术助手。

## 风格
- 精确但不冷酷
- 专业但不卖弄
- 简洁但不省略关键信息

## 边界
- 不提供非法建议
- 不执行危险操作
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 引导文件未生效 | 文件不存在或路径错误 | 检查 `workspace` 配置 |
| 提示词过大 | `bootstrapMaxChars` 太大 | 减小限制或精简文件 |
| Skills 不可用 | Skill 路径错误 | 检查 Skill 安装 |
| 时间不正确 | `userTimezone` 未设置 | 配置正确时区 |

### 调试命令

```bash
# 查看上下文使用
/context list

# 查看详细上下文
/context detail

# 检查工作区文件
ls -la ~/.openclaw/workspace/
```

## 🎯 最佳实践

### 引导文件管理

| 实践 | 说明 |
|------|------|
| **保持精简** | 引导文件应专注、简洁 |
| **定期更新** | 根据使用情况调整内容 |
| **使用版本控制** | 将工作区放入 Git |
| **分类清晰** | 不同类型信息放不同文件 |

### 提示词优化

| 实践 | 说明 |
|------|------|
| **稳定部分** | 保持固定内容以提升缓存 |
| **动态部分** | 仅包含必要的变化信息 |
| **子智能体** | 使用 `minimal` 模式减少开销 |
| **按需 Skills** | 避免加载所有 Skills |

### 缓存效率

