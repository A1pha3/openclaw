## 💡 为什么需要会话修剪？

### 类比：修剪就像"只保留需要的书"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    修剪类比解释                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  场景：你在图书馆研究一个主题                                           │
│                                                                         │
│  不修剪的情况：                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  桌子上堆了 100 本书                                            │   │
│  │  • 大部分是之前参考过的，现在已经不需要                         │   │
│  │  • 找当前需要的书很困难                                         │   │
│  │  • 每次都要浏览所有内容                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  修剪后：                                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  桌子上只保留最近参考的 10 本书                                 │   │
│  │  • 当前需要的书触手可及                                         │   │
│  │  • 旧书放回书架（需要时可取）                                   │   │
│  │  • 工作空间整洁高效                                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 会话修剪：                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  修剪前：上下文包含所有历史工具结果（可能数万 token）            │   │
│  │  修剪后：只保留最近的工具结果，旧结果被替换/清除                 │   │
│  │                                                                 │   │
│  │  效果：                                                          │   │
│  │  ✅ 减少 token 使用（降低成本）                                  │   │
│  │  ✅ 提高缓存命中率（Anthropic 缓存）                            │   │
│  │  ✅ 加快响应速度                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **修剪** | 每次请求前临时清理旧工具结果（内存中） |
| **压缩** | 总结对话并持久化到历史记录（磁盘） |
| **TTL** | 缓存有效期，超过后需要重新缓存 |
| **软修剪** | 保留工具结果的首尾部分 |
| **硬清除** | 完全替换工具结果为占位符 |

## 🔄 修剪 vs 压缩

### 关键区别

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    修剪 vs 压缩对比                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  修剪（Pruning）：                                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 对象：旧的工具结果                                          │   │
│  │  • 时机：每次 LLM 调用前                                       │   │
│  │  • 范围：仅内存（临时）                                        │   │
│  │  • 持久化：不修改历史记录                                      │   │
│  │  • 可逆：是（下次请求重新加载）                                │   │
│  │  • 目的：减少缓存写入成本                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  压缩（Compaction）：                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 对象：早期对话消息                                          │   │
│  │  • 时机：接近上下文窗口限制时                                  │   │
│  │  • 范围：磁盘历史记录（永久）                                  │   │
│  │  • 持久化：修改 *.jsonl 文件                                   │   │
│  │  • 可逆：否（历史被覆盖）                                      │   │
│  │  • 目的：管理上下文窗口大小                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 使用场景对比

| 场景 | 推荐方案 | 原因 |
|------|----------|------|
| 长会话，工具输出多 | 修剪 | 减少缓存成本 |
| 接近上下文限制 | 压缩 | 释放 token 空间 |
| 频繁交互 | 修剪 | 保持响应速度 |
| 需要长期记忆 | 压缩 | 持久化重要信息 |

## ⚙️ 运行机制

### 触发条件

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    修剪触发流程                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  会话空闲                                                             │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  检查：最后一次 Anthropic 调用是否超过 TTL？                   │   │
│  │                                                                 │   │
│  │  NO ──► 直接发送请求（不修剪）                                  │   │
│  │                                                                 │   │
│  │  YES ──► 执行修剪                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  1. 识别可修剪的工具结果                                        │   │
│  │  2. 应用软修剪或硬清除                                          │   │
│  │  3. 保护最近 N 条助手消息                                       │   │
│  │  4. 重置 TTL 缓存窗口                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│     │                                                                   │
│     ▼                                                                   │
│  发送修剪后的请求                                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 运行规则

| 规则 | 说明 |
|------|------|
| **仅 Anthropic** | 只对 Anthropic API 和 OpenRouter Anthropic 模型生效 |
| **仅工具结果** | 用户和助手消息永不修改 |
| **临时操作** | 不修改磁盘上的会话历史 |
| **TTL 重置** | 修剪后缓存窗口重新开始计时 |

## 🧠 智能默认值

### OAuth 或 Setup-Token 配置

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "1h"  // OAuth 用户默认 1 小时
      }
    }
  }
}
```

### API 密钥配置

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "30m"  // API 密钥用户默认 30 分钟
      },
      models: {
        providers: {
          anthropic: {
            models: {
              "*": {
                cacheControlTtl: "1h"  // 默认缓存 1 小时
              }
            }
          }
        }
      }
    }
  }
}
```

### 最佳 TTL 配置

| 场景 | 推荐 TTL | 原因 |
|------|----------|------|
| 频繁交互 | `"5m"` - `"15m"` | 保持缓存新鲜 |
| 间歇使用 | `"30m"` - `"1h"` | 平衡缓存和成本 |
| 偶尔使用 | `"1h"` - `"4h"` | 最大化缓存利用 |

## 🛠️ 可以剪枝的内容

### 修剪对象

| 可修剪 | 不可修剪 |
|--------|----------|
| ✅ `toolResult` 消息 | ❌ 用户消息 |
| ✅ 旧的工具结果 | ❌ 助手消息 |
| ✅ 超大工具输出 | ❌ 包含图像块的结果 |
| ✅ 指定工具的结果 | ❌ 最近 N 条助手后的结果 |

### 保护机制

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    助手消息保护                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  会话历史：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [用户] 请求 A                                                  │   │
│  │  [AI] 回复 A ─────────────────────────────────┐                  │   │
│  │  [工具] 结果 A（可修剪）                     │                  │   │
│  │  [用户] 请求 B                               │                  │   │
│  │  [AI] 回复 B ─────────────────────────────┐   │                  │   │
│  │  [工具] 结果 B（可修剪）                 │   │                  │   │
│  │  [用户] 请求 C                           │   │                  │   │
│  │  [AI] 回复 C ─────────────────────┐       │   │                  │   │
│  │  [工具] 结果 C（可修剪）         │       │   │                  │   │
│  │  [用户] 当前请求                │       │   │                  │   │
│  │  [AI] 当前回复 ◄────────────────┘       │   │ ← 保护点         │   │
│  │                                        │   │   （keepLastAssistants = 3）│
│  └────────────────────────────────────────┴───┘   │                  │
│                                                  │                  │
│  保护线之后的结果不会被修剪                       │                  │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📐 上下文窗口估算

### 解析顺序

```
上下文窗口大小解析（按优先级）：

1. models.providers.*.models[].contextWindow 覆盖
   │
   ├─ 未设置 ──► 2. 模型定义的 contextWindow（注册表）
   │              │
   │              ├─ 未定义 ──► 3. 默认 200000 token
   │
4. agents.defaults.contextTokens（上限）
```

### 字符到 Token 估算

```
字符数 ≈ Token × 4

示例：
- 100,000 字符 ≈ 25,000 token
- 200,000 字符 ≈ 50,000 token
```

## 🔧 修剪模式

### cache-ttl 模式

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "5m"
      }
    }
  }
}
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `mode` | `"off"` | `"off"` \| `"cache-ttl"` |
| `ttl` | `"5m"` | 触发修剪的空闲时间 |

### off 模式

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "off"
      }
    }
  }
}
```

完全禁用修剪。

## ✂️ 软修剪 vs 硬清除

### 软修剪（Soft Trim）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    软修剪示意                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  原始工具结果（50,000 字符）：                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [大量输出内容...]                                              │   │
│  │                                                                 │   │
│  │  （中间有 45,000 字符）                                         │   │
│  │                                                                 │   │
│  │  [...更多输出...]                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  软修剪后（保留头部和尾部）：                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [前 1500 字符的输出...]                                       │   │
│  │                                                                 │   │
│  │  ... (46,000 chars trimmed, original: 50000 chars)             │   │
│  │                                                                 │   │
│  │  [...后 1500 字符的输出]                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**配置参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `softTrimRatio` | `0.3` | 超过此比例才修剪 |
| `maxChars` | `4000` | 修剪后最大字符数 |
| `headChars` | `1500` | 保留的头部字符数 |
| `tailChars` | `1500` | 保留的尾部字符数 |

### 硬清除（Hard Clear）

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    硬清除示意                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  原始工具结果：                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [大量输出内容...]                                              │   │
│  │  [...更多输出...]                                              │   │
│  │  [...持续输出...]                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  硬清除后：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [Old tool result content cleared]                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

**配置参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `hardClearRatio` | `0.5` | 超过此比例才清除 |
| `enabled` | `true` | 是否启用硬清除 |
| `placeholder` | `"[Old tool result content cleared]"` | 替换文本 |

## 🔧 工具选择

### 配置允许/拒绝列表

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        tools: {
          allow: ["exec", "read", "write"],
          deny: ["*image*", "screenshot"]
        }
      }
    }
  }
}
```

### 匹配规则

| 规则 | 说明 |
|------|------|
| `*` 通配符 | 匹配任意字符 |
| 拒绝优先 | `deny` 优先于 `allow` |
| 不区分大小写 | `"EXEC"` 匹配 `"exec"` |
| 空允许列表 | 允许所有工具 |

### 配置示例

```json5
// 仅修剪特定工具
{
  tools: {
    allow: ["exec", "grep"]  // 只修剪这两个工具的结果
  }
}

// 排除图像相关工具
{
  tools: {
    deny: ["*image*", "screenshot", "analyze-image"]
  }
}

// 修剪所有工具（默认）
{
  tools: {
    allow: []  // 空列表 = 允许所有
  }
}
```

## 📊 完整配置示例

### 基础配置

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "off"
      }
    }
  }
}
```

### TTL 感知修剪

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "5m",
        keepLastAssistants: 3
      }
    }
  }
}
```

### 自定义修剪参数

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "10m",
        keepLastAssistants: 5,
        softTrimRatio: 0.25,
        hardClearRatio: 0.6,
        minPrunableToolChars: 30000,
        softTrim: {
          maxChars: 5000,
          headChars: 2000,
          tailChars: 2000
        },
        hardClear: {
          enabled: true,
          placeholder: "[Previous output removed to save context]"
        }
      }
    }
  }
}
```

### 工具级别配置

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        ttl: "5m",
        tools: {
          allow: ["exec", "bash", "read-file"],
          deny: ["*image*", "multi-modal"]
        }
      }
    }
  }
}
```

## 🎯 最佳实践

### 何时启用修剪

| 场景 | 推荐 | 原因 |
|------|------|------|
| 频繁使用会话 | ✅ 启用 | 减少缓存成本 |
| 偶尔使用 | ❌ 禁用 | 缓存自然过期 |
| 工具输出多 | ✅ 启用 | 显著减少 token |
| 图像处理多 | ⚠️ 谨慎 | 图像结果不修剪 |

### TTL 设置建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    TTL 选择指南                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  高频交互（每几分钟一次）                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  TTL: 5m - 15m                                                 │   │
│  │  • 保持缓存新鲜                                                │   │
│  │  • 避免频繁修剪                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  中频交互（每几小时一次）                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  TTL: 30m - 1h                                                 │   │
│  │  • 平衡缓存命中和成本                                          │   │
│  │  • 与 cacheControlTtl 匹配                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  低频交互（每天一次或更少）                                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  TTL: 1h - 4h 或禁用                                           │   │
│  │  • 最大化缓存利用                                              │   │
│  │  • 修剪可能不如直接缓存                                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 与缓存控制配合

```json5
{
  agents: {
    defaults: {
      contextPruning: {
        ttl: "5m"  // 应 ≤ cacheControlTtl
      },
      models: {
        providers: {
          anthropic: {
            models: {
              "*": {
                cacheControlTtl: "5m"  // 与修剪 TTL 匹配
              }
            }
          }
        }
      }
    }
  }
}
```

### 性能优化

| 实践 | 效果 |
|------|------|
| `ttl` ≤ `cacheControlTtl` | 最佳缓存命中率 |
| 增加 `keepLastAssistants` | 保留更多上下文 |
| 设置 `minPrunableToolChars` | 避免修剪小结果 |
