## 💡 为什么需要路由？

### 类比：路由就像"分诊台"

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    分诊台类比                                             │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：医院分诊                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  患者 A（心脏问题）────► 分诊护士 ──► 心内科                     │   │
│  │  患者 B（骨折）──────► 分诊护士 ──► 骨科                         │   │
│  │  患者 C（感冒）──────► 分诊护士 ──► 普通内科                     │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 专业处理                                                      │   │
│  │  • 各科独立记录                                                │   │
│  │  • 避免交叉混乱                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  OpenClaw 场景：消息路由                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  WhatsApp 群组 ──► 路由系统 ──► Support 代理                    │   │
│  │  Telegram 私聊 ──► 路由系统 ──► Personal 代理                   │   │
│  │  Discord 频道 ──► 路由系统 ──► Community 代理                   │   │
│  │                                                                 │   │
│  │  好处：                                                          │   │
│  │  • 不同智能体处理不同场景                                       │   │
│  │  • 独立的会话记忆                                             │   │
│  │  • 可配置的工作区                                             │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心原则

| 原则 | 说明 |
|------|------|
| **确定性路由** | 路由规则配置驱动，模型不选择 |
| **自动回源** | 回复自动发送回消息来源的频道 |
| **会话隔离** | 不同代理有独立的会话存储 |

## 📋 关键术语

### 术语定义

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       路由相关术语                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Channel（渠道）                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  消息平台：whatsapp, telegram, discord, slack, signal,          │   │
│  │           imessage, webchat                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  AccountId（账户 ID）                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  每个渠道的账户实例                                              │   │
│  │  示例：telegram:account1, whatsapp:business                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  AgentId（代理 ID）                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  隔离的工作区 + 会话存储                                         │   │
│  │  示例：main, support, personal, logger                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  SessionKey（会话密钥）                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用于存储上下文和控制并发的桶密钥                                │   │
│  │  格式：agent:<agentId>:<其他标识>                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🔑 会话密钥结构

### 密钥格式

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       会话密钥结构                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  直接消息（DM）──────────────────────────────────────────────────────► │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  agent:<agentId>:<mainKey>                                      │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  • agent:main:main         # 主代理的主会话                     │   │
│  │  • agent:support:main      # 支持代理的主会话                   │   │
│  │                                                                 │   │
│  │  特点：所有私聊消息折叠到主会话                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  群组 ──────────────────────────────────────────────────────────────► │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  agent:<agentId>:<channel>:group:<id>                           │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  • agent:main:telegram:group:-1001234567890                     │   │
│  │  • agent:support:whatsapp:group:120363403215116621@g.us         │   │
│  │                                                                 │   │
│  │  特点：每个群组独立会话                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  频道/房间 ────────────────────────────────────────────────────────► │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  agent:<agentId>:<channel>:channel:<id>                         │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  • agent:main:discord:channel:1234567890                        │   │
│  │  • agent:support:slack:channel:C12345678                        │   │
│  │                                                                 │   │
│  │  特点：每个频道/房间独立会话                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  线程（Discord/Slack）───────────────────────────────────────────────► │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  <基础密钥>:thread:<threadId>                                   │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  • agent:main:discord:channel:123:thread:987654                 │   │
│  │  • agent:support:slack:channel:C123:thread:456789               │   │
│  │                                                                 │   │
│  │  特点：线程独立于父频道                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  话题（Telegram 论坛）────────────────────────────────────────────────► │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  agent:<agentId>:telegram:group:<id>:topic:<topicId>            │   │
│  │                                                                 │   │
│  │  示例：                                                          │   │
│  │  • agent:main:telegram:group:-1001234567890:topic:42            │   │
│  │                                                                 │   │
│  │  特点：话题独立于父群组                                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🎯 路由规则

### 代理选择优先级

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       代理选择优先级（从高到低）                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 精确对等匹配 ──────────────────────────────────────────────────► │
│     │                                                                   │
│     │  bindings 配置了 peer.kind + peer.id                             │
│     │  示例：{ match: { channel: "telegram", peer: { kind: "group",    │
│     │                                            id: "-100123" } },    │
│     │         agentId: "support" }                                     │
│     │                                                                   │
│  2. 公会匹配（Discord）─────────────────────────────────────────────► │
│     │                                                                   │
│     │  通过 guildId 匹配                                               │
│     │  示例：{ match: { channel: "discord", guildId: "123456" },       │
│     │         agentId: "community" }                                   │
│     │                                                                   │
│  3. 团队匹配（Slack）───────────────────────────────────────────────► │
│     │                                                                   │
│     │  通过 teamId 匹配                                                │
│     │  示例：{ match: { channel: "slack", teamId: "T123" },            │
│     │         agentId: "work" }                                        │
│     │                                                                   │
│  4. 账户匹配 ─────────────────────────────────────────────────────► │
│     │                                                                   │
│     │  通过特定 accountId 匹配                                         │
│     │  示例：{ match: { channel: "whatsapp",                          │
│     │                     accountId: "business" },                     │
│     │         agentId: "sales" }                                       │
│     │                                                                   │
│  5. 渠道匹配 ─────────────────────────────────────────────────────► │
│     │                                                                   │
│     │  匹配该渠道的任何账户                                             │
│     │  示例：{ match: { channel: "telegram" }, agentId: "main" }        │
│     │                                                                   │
│  6. 默认代理 ─────────────────────────────────────────────────────► │
│                                                                         │
│     agents.list[].default 设置的代理                                    │
│     否则使用第一个列表条目                                               │
│     回退到 "main"                                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📢 广播组

### 多代理并行

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       广播组机制                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  普通模式（单代理）                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  入站消息 ──► 路由到单个代理 ──► 单个回复                         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  广播模式（多代理）                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  入站消息                                                         │   │
│  │     │                                                             │   │
│  │     ├─────────────► Alfred 代理 ──► 回复 1                       │   │
│  │     │                                                             │   │
│  │     ├─────────────► Baerbel 代理 ──► 回复 2                      │   │
│  │     │                                                             │   │
│  │     └─────────────► Charles 代理 ──► 回复 3                      │   │
│  │                                                                 │   │
│  │  结果：多个代理并行处理，各自回复                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  使用场景：                                                              │
│  • 需要多个专家同时分析                                                │
│  • 对比不同代理的输出                                                   │
│  • 日志记录 + 正常处理并行                                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 配置示例

```json5
{
  broadcast: {
    strategy: "parallel",    // 并行执行
    // 群组 ID → 代理列表
    "120363403215116621@g.us": ["alfred", "baerbel"],
    "+15555550123": ["support", "logger"],
    "-1001234567890": ["agent1", "agent2", "agent3"]
  }
}
```

| 选项 | 说明 |
|------|------|
| `strategy: "parallel"` | 并行执行所有代理 |
| `strategy: "first"` | 第一个响应的代理返回结果 |
| 对等体 ID | 消息来源的群组或用户 ID |

## ⚙️ 配置示例

### 基本配置

```json5
{
  agents: {
    list: [
      {
        id: "main",
        name: "Main",
        workspace: "~/.openclaw/workspace"
      },
      {
        id: "support",
        name: "Support Team",
        workspace: "~/.openclaw/workspace-support",
        default: true
      },
      {
        id: "sales",
        name: "Sales Bot",
        workspace: "~/.openclaw/workspace-sales"
      }
    ]
  },
  bindings: [
    // Slack 工作区 → Support 代理
    { match: { channel: "slack", teamId: "T123456" }, agentId: "support" },

    // Telegram 特定群组 → Sales 代理
    { match: { channel: "telegram", peer: { kind: "group", id: "-1001234567890" } }, agentId: "sales" },

    // WhatsApp 商业账户 → Sales 代理
    { match: { channel: "whatsapp", accountId: "business" }, agentId: "sales" },

    // Discord 公会 → Community 代理
    { match: { channel: "discord", guildId: "1234567890" }, agentId: "community" },

    // 所有 Telegram 消息 → Main 代理
    { match: { channel: "telegram" }, agentId: "main" }
  ]
}
```

### 复杂场景配置

```json5
{
  agents: {
    list: [
      { id: "personal", workspace: "~/.openclaw/personal" },
      { id: "work", workspace: "~/.openclaw/work" },
      { id: "support", workspace: "~/.openclaw/support", default: true }
    ]
  },
  bindings: [
    // 工作相关 Slack → Work 代理
    { match: { channel: "slack", teamId: "T_WORK" }, agentId: "work" },

    // 支持群组 → Support 代理（精确匹配）
    { match: { channel: "telegram", peer: { kind: "group", id: "-100_SUPPORT" } }, agentId: "support" },

    // 个人 Telegram → Personal 代理（默认）
    { match: { channel: "telegram" }, agentId: "personal" }
  ],
  broadcast: {
    // 监控群组：多个代理并行
    "-100_MONITOR": ["support", "logger", "analytics"]
  }
}
```

## 💾 会话存储

### 存储位置

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       会话存储结构                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ~/.openclaw/                                                            │
│  ├── agents/                                                            │
│  │   ├── main/                                                          │
│  │   │   └── sessions/                                                 │
│  │   │       ├── sessions.json ───► 会话索引                           │
│  │   │       └── *.jsonl ────────► 消息历史                           │
│  │   │                                                                 │
│  │   ├── support/                                                       │
│  │   │   └── sessions/                                                 │
│  │   │       ├── sessions.json                                         │
│  │   │       └── *.jsonl                                               │
│  │   │                                                                 │
│  │   └── sales/                                                         │
│  │       └── sessions/                                                 │
│  │           ├── sessions.json                                         │
│  │           └── *.jsonl                                               │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 自定义存储路径

```json5
{
  agents: {
    list: [
      {
        id: "custom",
        // 自定义工作区路径
        workspace: "/path/to/custom/workspace",
        // 或通过配置覆盖
        "session.store": "/path/to/sessions"
      }
    ]
  }
}
```

### 存储模板

支持 `{agentId}` 模板变量：

```json5
{
  // 所有代理共享的基础目录
  "session.store": "/data/openclaw/{agentId}/sessions"
}
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 消息路由到错误的代理 | 绑定规则优先级错误 | 检查绑定顺序和匹配条件 |
| 会话混乱 | 多个代理共享会话 | 使用不同的 `agentId` |
| 群组消息不回复 | 代理配置了 `restrict` 规则 | 检查权限配置 |
| WebChat 显示空 | 代理没有活动会话 | 发送测试消息激活 |

# 查看当前代理配置

# 查看会话状态
openclaw sessions list

## 🎯 最佳实践

### 配置建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                       路由配置建议                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  1. 按场景划分代理                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • Personal 代理：个人聊天                                       │   │
│  │  • Work 代理：工作相关渠道                                       │   │
│  │  • Support 代理：客户支持                                        │   │
│  │                                                                 │   │
│  │  好处：上下文隔离，职责清晰                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  2. 使用精确匹配优先                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 先匹配具体的群组/频道                                         │   │
│  │  • 再匹配渠道                                                    │   │
│  │  • 最后使用默认代理                                              │   │
│  │                                                                 │   │
│  │  好处：精细控制，避免意外路由                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  3. 为每个代理设置独立工作区                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 不同工作区避免会话混乱                                        │   │
│  │  • 独立的配置和设置                                              │   │
│  │  • 更好的故障隔离                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  4. 设置默认代理                                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 在一个代理上设置 `default: true`                              │   │
│  │  • 处理未匹配的消息                                              │   │
│  │  • 避免消息丢失                                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 绑定规则排序

```json5
{
  bindings: [
    // 1. 最精确的匹配（特定群组）
    { match: { channel: "telegram", peer: { kind: "group", id: "-123" } }, agentId: "support" },

    // 2. 平台特定匹配（Discord 公会）
    { match: { channel: "discord", guildId: "456" }, agentId: "community" },

    // 3. 账户匹配
    { match: { channel: "whatsapp", accountId: "business" }, agentId: "sales" },

    // 4. 渠道匹配（最宽泛）
    { match: { channel: "telegram" }, agentId: "main" }
  ]
}
```
