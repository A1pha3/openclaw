## 💡 为什么需要压缩？

### 类比：压缩就像整理笔记本

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    压缩类比                                            │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  不压缩的混乱笔记本：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  第 1 页 ─┐                                                        │   │
│  │  第 2 页 ├──► ... 100 页                                              │   │
│  │  第 3 页 ─┘                                                        │   │
│  │                                                                 │   │
│  │  问题：                                                          │   │
│  │  • 翻找旧信息很困难                                              │   │
│  │  • Token 限制（模型无法读取全部）                              │   │
│  │  • 成本高昂（每次都发送全部历史）                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  压缩后的笔记本：                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  摘要 1：第 1-80 页的内容                                         │   │
│  │  摘要 2：第 81-90 页的决策                                        │   │
│  │  摘要 3：近期消息（第 91-100 页）                                 │   │
│  │                                                                 │   │
│  │  优势：                                                          │   │
│  │  ✓ 快速找到历史信息                                              │   │
│  │  ✓ Token 高效利用                                               │   │
│  │  ✓ 保持对话连续性                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心概念

| 概念 | 说明 |
|------|------|
| **上下文窗口** | 模型可见的最大 token 数 |
| **压缩** | 总结早期对话为摘要，持久化到 JSONL |
| **修剪** | 仅裁剪旧工具结果（内存中） |
| **记忆刷新** | 压缩前写入持久笔记 |

## 🔄 压缩机制

### 什么是压缩？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    压缩前后对比                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  压缩前：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [用户] 你好                                                      │   │
│  │  [AI] 你好！有什么我可以帮助的？                              │   │
│  │  [用户] 帮我写一个 Python 函数                                   │   │
│  │  [AI] 好的...（函数代码）                                           │   │
│  │  [用户] 再添加一个功能                                             │   │
│  │  [AI] 好的...（函数代码）                                           │   │
│  │  ...（继续 100 轮）                                              │   │
│  │  [用户] 之前那个函数的参数是什么？                                │   │
│  │  [AI] 需要向前翻 100 页... ❌ Token 超限！                     │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  压缩后：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  🧹 摘要：用户请求帮助编写和优化 Python 函数                       │   │
│  │     - 讨论了参数设计和错误处理                                    │   │
│  │     - 最终实现了函数 with 语句和日志记录                           │   │
│  │  ──────────────────────────────────────────────────────────      │   │
│  │  [用户] 之前那个函数的参数是什么？                                │   │
│  │  [AI] 根据摘要，第一个参数是 input_file，第二个是... ✅        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 压缩特性

| 特性 | 说明 |
|------|------|
| **持久化** | 摘要存储在 JSONL 历史中 |
| **保持连续性** | 保留近期消息不变 |
| **可重试** | 使用压缩后上下文重试原始请求 |

## ⚙️ 自动压缩

### 触发条件

当会话接近或超过上下文窗口时，自动触发：

```
上下文使用 > 85% → 触发自动压缩
```

### 压缩前记忆刷新

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    压缩前记忆刷新                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  会话接近限制 ──► 触发记忆刷新 ──► AI 写入持久笔记 ──► 压缩执行      │
│                      │              │                        │          │
│                      │              │  ┌─────────────────┐  │          │
│                      │              │  │ 写入记忆文件    │  │          │
│                      │              │  └─────────────────┘  │          │
│                      │              │                        │          │
│                      ▼              ▼                        ▼          │
│                 静默回合（NO_REPLY）    压缩历史记录                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 配置记忆刷新

```json5
{
  agents: {
    defaults: {
      compaction: {
        reserveTokensFloor: 20000,
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000,
          systemPrompt: "Session nearing compaction. Store durable memories now.",
          prompt: "Write any lasting notes to memory/YYYY-MM-DD.md; reply with NO_REPLY if nothing to store.",
        },
      },
    },
  },
}
```

### 自动压缩指示

- **详细模式**：显示 `🧹 Auto-compaction complete`
- **状态查询**：`/status` 显示 `🧹 Compactions: <count>`

## 🛠️ 手动压缩

### 触发压缩

```
/compact Focus on decisions and open questions
```

### 压缩指令选项

| 指令 | 说明 |
|------|------|
| **无指令** | 压缩所有内容 |
| **指定主题** | 聚焦于特定主题 |
| **保留关键字** | 保留关键信息 |

### 使用场景

| 场景 | 命令 |
|------|------|
| 过时上下文 | `/compact` |
| 聚焦主题 | `/compact Focus on X` |
| 新阶段开始 | `/compact Starting new phase` |

## 📐 配置参数

### 压缩配置

```json5
{
  agents: {
    defaults: {
      compaction: {
        // 保留 token 下限
        reserveTokensFloor: 20000,

        // 自动压缩阈值（百分比）
        autoThreshold: 0.85,

        // 压缩模式
        mode: "adaptive",  // adaptive | eager | lazy

        // 记忆刷新配置
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000,
        },
      },
    },
  },
}
```

### 模式对比

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **adaptive** | 智能判断何时压缩 | 默认推荐 |
| **eager** | 更早压缩 | 重要对话 |
| **lazy** | 更晚压缩 | 随意聊天 |

## 🔍 压缩 vs 修剪

### 关键区别

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    压缩 vs 修剪对比                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  压缩（Compaction）：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 总结早期对话为摘要                                             │   │
│  │  • 持久化到 JSONL 历史记录                                      │   │
│  │  • 不可逆（历史记录被修改）                                    │   │
│  │  • 用于管理上下文窗口                                          │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  修剪（Pruning）：                                                     │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  • 裁剪旧的工具结果                                              │   │
│  │  • 仅在内存中生效（每个请求）                                    │   │
│  │  • 不修改历史记录                                                │   │
│  │  • 用于减少单次请求的 token 消耗                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 配置修剪

```json5
{
  agents: {
    defaults: {
      // 修剪工具结果
      pruneTools: true,

      // 修剪保留
      pruneKeepRecent: 3,

      // 最大 token 数
      maxToolTokens: 10000,
    },
  },
}
```
