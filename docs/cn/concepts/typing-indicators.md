## 💡 什么是输入指示器？

### 类比：输入指示器就像"请稍候"牌

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    输入指示器类比                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  现实场景：办公室门牌                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  [请稍候，正在处理...]                                           │   │
│  │                                                                 │   │
│  │  作用：                                                          │   │
│  │  • 告诉访客有人在里面                                          │   │
│  │  • 管理等待预期                                                │   │
│  │  • 提供反馈表明系统在运行                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  聊天应用场景：                                                          │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户 ──► [你的问题...]                                         │   │
│  │                                                                 │   │
│  │  AI [正在输入...] ◀◀◀ ──► [回复内容]                          │   │
│  │                                                                 │   │
│  │  作用：                                                          │   │
│  │  • 告诉用户 AI 正在思考                                        │   │
│  │  • 管理用户等待预期                                            │   │
│  │  • 提供即时反馈                                                │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 核心价值

| 价值 | 说明 |
|------|------|
| **即时反馈** | 用户立即知道 AI 已收到消息 |
| **等待管理** | "正在输入"比静默等待体验更好 |
| **状态通知** | 表明系统正在处理请求 |

## 🎨 显示模式

### 模式对比

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    显示模式对比                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  never（从不显示）                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户消息 ──► [静默] ────────────────────────► AI 回复          │   │
│  │                                                                 │   │
│  │  适用：后台任务、自动化场景                                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  message（消息开始时）                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户消息 ──► [静默] ──► [正在输入...] ──► AI 回复             │   │
│  │                           ↑                                      │   │
│  │                           首个文本增量                            │   │
│  │                                                                 │   │
│  │  适用：大多数场景（默认）                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  thinking（思考开始时）                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户消息 ──► [正在输入...] ──► AI 回复                         │   │
│  │                      ↑                                          │   │
│  │                      首个推理增量                                │   │
│  │                                                                 │   │
│  │  适用：需要推理级别可见的场景                                   │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  instant（立即显示）                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户消息 ──► [正在输入...] ──► AI 回复                         │   │
│  │                      ↑                                          │   │
│  │                      模型循环开始                                │   │
│  │                                                                 │   │
│  │  适用：需要最快反馈的场景                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 触发时机顺序

```
从晚到早（显示越来越早）：

never ──► message ──► thinking ──► instant
 (从不)    (消息开始)  (思考开始)   (立即)

用户体验：
instant  >  thinking >  message >  never
(最快反馈)                   (无反馈)
```

## ⚙️ 默认行为

### 未设置 typingMode 时

当 `agents.defaults.typingMode` 未设置时，使用旧版行为：

| 场景 | 行为 |
|------|------|
| **私聊** | 模型循环开始后立即显示 |
| **群聊中被提及** | 立即显示输入指示器 |
| **群聊中未被提及** | 消息开始流式传输时显示 |
| **心跳运行** | 不显示输入指示器 |

### 默认行为图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    默认行为决策                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  收到消息                                                                │
│     │                                                                   │
│     ▼                                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  是私聊？                                                        │   │
│  │                                                                 │   │
│  │  YES ──► 立即显示 [正在输入...]                                   │   │
│  │                                                                 │   │
│  │  NO ──► 是群聊且被提及？                                       │   │
│  │          │                                                       │   │
│  │          YES ──► 立即显示 [正在输入...]                          │   │
│  │          │                                                       │   │
│  │          NO ──► 等待消息开始流式传输                            │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 🔧 配置详解

### 基础配置

```json5
{
  agents: {
    defaults: {
      typingMode: "thinking",         // 显示模式
      typingIntervalSeconds: 6          // 刷新间隔（秒）
    }
  }
}
```

### 模式详解

| 模式 | 触发条件 | 说明 |
|------|----------|------|
| `never` | 不触发 | 永远不显示输入指示器 |
| `instant` | 模型循环开始 | 立即显示，即使最终是静默回复 |
| `thinking` | 第一个推理增量 | 需要 `reasoningLevel: "stream"` |
| `message` | 第一个文本增量 | 忽略 `NO_REPLY` 静默令牌 |

### 刷新频率

```
typingIntervalSeconds 控制输入指示器的刷新频率

• 默认值：6 秒
• 作用：保持"正在输入"状态活跃
• 不影响：开始显示的时间
```

### 会话覆盖

```json5
// 按会话覆盖
{
  session: {
    typingMode: "message",
    typingIntervalSeconds: 4
  }
}
```

### 支持的渠道

| 渠道 | 输入指示器 | 说明 |
|------|-----------|------|
| **Telegram** | ✅ | "正在输入..." 状态 |
| **Discord** | ✅ | "正在输入..." 状态 |
| **Slack** | ✅ | 队列中的输入活动 |
| **WhatsApp** | ❌ | 不支持 API |

## ⚠️ 注意事项

### 特殊情况

| 情况 | 行为 | 说明 |
|------|------|------|
| 纯静默回复 | 不显示 | `message` 模式下 `NO_REPLY` 不触发 |
| 无推理增量 | 不显示 | `thinking` 模式需要有推理内容 |
| 心跳运行 | 不显示 | 无论何种模式都不显示 |
| 群组未被提及 | 延迟显示 | 等待消息开始流式传输 |

### reasoningLevel 依赖

```
thinking 模式需要推理级别流式传输：

{
  agents: {
    defaults: {
      reasoningLevel: "stream",  // 启用推理流式传输
      typingMode: "thinking"      // 可在推理增量时显示
    }
  }
}
```

## 🎯 配置示例

### 标准配置

```json5
{
  agents: {
    defaults: {
      typingMode: "message",
      typingIntervalSeconds: 6
    }
  }
}
```

### 快速反馈配置

```json5
{
  agents: {
    defaults: {
      typingMode: "instant",           // 立即显示
      typingIntervalSeconds: 4          // 更快刷新
    }
  }
}
```

### 推理可见配置

```json5
{
  agents: {
    defaults: {
      reasoningLevel: "stream",         // 启用推理流式传输
      typingMode: "thinking",           // 推理时显示
      typingIntervalSeconds: 6
    }
  }
}
```

### 禁用指示器

```json5
{
  agents: {
    defaults: {
      typingMode: "never"               // 完全禁用
    }
  }
}
```

### 后台任务配置

```json5
{
  agents: {
    list: [
      {
        id: "background",
        typingMode: "never"              // 后台任务不显示
      }
    ]
  }
}
```

## 🐛 故障排查

### 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 指示器不显示 | `typingMode: "never"` | 更改为其他模式 |
| `thinking` 无效 | 未设置 `reasoningLevel` | 添加 `reasoningLevel: "stream"` |
| 指示器闪烁 | `typingIntervalSeconds` 太小 | 增加刷新间隔 |

### 调试技巧

```bash
# 启用详细日志
openclaw gateway --verbose

# 查看输入指示器日志
grep "typing" /tmp/openclaw/openclaw-*.log

# 检查配置
openclaw config get agents.defaults.typingMode
```

## 🎯 最佳实践

### 场景推荐

| 场景 | 推荐模式 | 刷新间隔 |
|------|----------|----------|
| **个人助手** | `message` | `6` 秒 |
| **快速响应** | `instant` | `4` 秒 |
| **推理可见** | `thinking` | `6` 秒 |
| **后台任务** | `never` | - |

### 刷新间隔建议

| 间隔 | 效果 | 适用场景 |
|------|------|----------|
| `3-4` 秒 | 更频繁的刷新 | 快速响应 |
| `6` 秒 | 标准（默认） | 大多数场景 |
| `10+` 秒 | 较少刷新 | 长时间处理 |
