| **可扩展性** | 通过钩子系统支持自定义扩展 |

## 📋 核心概念

### 什么是智能体循环？

智能体循环是智能体的**完整"真实"运行**路径：

```
接收 → 上下文组装 → 模型推理 → 工具执行 → 流式回复 → 持久化
```

**关键特性**：

| 特性 | 说明 |
|------|------|
| **序列化运行** | 每个会话的单次序列化运行 |
| **事件驱动** | 发出生命周期和流事件 |
| **状态管理** | 保持会话状态的一致性 |
| **超时控制** | 防止无限期运行 |

## 🔀 队列与并发控制

### 为什么需要队列？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    队列控制的必要性                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  无队列控制的混乱场景：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  用户 A 发送消息 ──┐                                              │   │
│  │  用户 B 发送消息 ──┼──► 同一会话 → ???                            │   │
│  │  用户 C 发送消息 ──┘                                              │   │
│  │                                                                 │   │
│  │  问题：                                                          │   │
│  │  • 消息竞争（谁先处理？）                                        │   │
│  │  • 会话历史混乱（顺序错误）                                      │   │
│  │  • 工具冲突（同时写文件）                                        │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  有队列控制的有序场景：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  消息 1 ──► 队列 ──► 处理 ──► 响应 1                             │   │
│  │  消息 2 ──►   │         │         │                             │   │
│  │  消息 3 ──►   │         │         │                             │   │
│  │                │         │         │                             │   │
│  │  ✓ 顺序保证                                                    │   │
│  │  ✓ 会话一致                                                    │   │
│  │  ✓ 资源隔离                                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 队列机制

| 特性 | 说明 |
|------|------|
| **会话通道序列化** | 按会话键（sessionKey）序列化 |
| **全局通道** | 可选的全局序列化 |
| **队列模式** | collect/steer/followup |
| **防止竞争** | 工具/会话竞争控制 |

## 🔌 钩子系统

### 两类钩子

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        钩子系统架构                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  内部钩子（Gateway 钩子）                                            │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  事件驱动脚本                                                     │   │
│  │  • agent:bootstrap（引导文件构建时）                            │   │
│  │  • 命令钩子（/new、/reset、/stop）                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  插件钩子（扩展点）                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │  智能体生命周期钩子：                                             │   │
│  │  • before_agent_start（运行前注入上下文）                        │   │
│  │  • agent_end（完成后检查元数据）                                │   │
│  │                                                                 │   │
│  │  工具钩子：                                                       │   │
│  │  • before_tool_call / after_tool_call（拦截工具）               │   │
│  │  • tool_result_persist（转换工具结果）                          │   │
│  │                                                                 │   │
│  │  消息钩子：                                                       │   │
│  │  • message_received / message_sending / message_sent            │   │
│  │                                                                 │   │
│  │  会话钩子：                                                       │   │
│  │  • session_start / session_end                                  │   │
│  │                                                                 │   │
│  │  Gateway 钩子：                                                   │   │
│  │  • gateway_start / gateway_stop                                  │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 内部钩子

| 钩子 | 时机 | 用途 |
|------|------|------|
| `agent:bootstrap` | 系统提示最终确定前 | 修改引导文件 |
| 命令钩子 | 命令触发时 | 自定义命令行为 |

### 插件钩子

| 钩子 | 时机 | 用途 |
|------|------|------|
| `before_agent_start` | 运行开始前 | 注入上下文 |
| `agent_end` | 完成后 | 检查元数据 |
| `before_compaction` / `after_compaction` | 压缩周期 | 观察/注释压缩 |
| `before_tool_call` / `after_tool_call` | 工具调用 | 拦截参数/结果 |
| `message_received` / `message_sending` / `message_sent` | 消息流转 | 处理消息 |

## 📡 事件流

### 事件类型

| 流 | 事件 | 说明 |
|----|------|------|
| **lifecycle** | `start` \| `end` \| `error` | 运行生命周期 |
| **assistant** | 增量流式传输 | 助手回复 |
| **tool** | `start` \| `update` \| `end` | 工具执行 |

### 生命周期事件

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    生命周期事件流                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  start ─────► [运行中] ─────► end                                   │
│                   │                                                 │
│                   │                                                 │
│                   ▼                                                 │
│                error（可选）                                       │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## ⏱️ 超时控制

### 两级超时

| 超时类型 | 默认值 | 作用范围 |
|----------|--------|----------|
| **agent.wait** | 30 秒 | 仅等待时间 |
| **运行时超时** | 600 秒 | 完整运行时间 |

### 超时配置

```json5
{
  agents: {
    defaults: {
      timeoutSeconds: 600,  // 智能体运行超时
    },
  },
}
```
